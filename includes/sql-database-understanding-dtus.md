資料庫輸送量單位 (DTU) 是 SQL Database 中的衡量單位，表示在指定驅動資料庫的硬體限制時，資料庫用來處理交易的相對能力。DTU 的概念類似於車輛上的馬力。它會說明資料庫之間的相對效能。就像車輛上的馬力，資料庫具有的 DTU 越多，它具有的功能就越強大。馬力是具備許多不同因數的函式：重量、引擎容量、渦輪增壓、排氣系統。同理，DTU 是資料庫可取用之記憶體、運算及磁碟數量與類型的函式。您可以在[基準測試概觀](https://msdn.microsoft.com/library/azure/dn741327.aspx)中閱讀我們如何判斷 DTU 的詳細資料)。

範例：基本資料庫有 5 個 DTU，這表示針對交易輸送量，它所具有的容量相對來說遠低於具有 1750 個 DTU 的高階 P11 資料庫。

![不同層級和等級的單一資料庫 DTU](./media/sql-database-understanding-dtus/single_db_dtus.png)

單一資料庫的 DTU 就相當於彈性資料庫的 eDTU。舉例來說，基本彈性資料庫集區中的資料庫最多可提供 5 個 eDTU。這與單一基本資料庫的效能相同。差別在於彈性資料庫不會取用集區中的任何 eDTU，除非有必要。

![不同層級的彈性集區](./media/sql-database-understanding-dtus/sqldb_elastic_pools.png)

舉個簡單的範例對您會有所幫助。取得有 1000 DTU 的基本彈性資料庫集區，並卸除其中 800 個資料庫。在任何時間點，只要 800 個資料庫中僅有 200 個正在使用 (5 DTU X 200 = 1000)，您就不會達到集區容量的限制，而且資料庫效能也不會降低。這個範例是為了清楚起見而簡化。實際計算時會稍微複雜一點。入口網站會替您做好這項工作，並根據資料庫使用量歷程記錄提供建議。若要了解此建議的運作方式，或自行進行計算，請參閱[彈性資料庫集區的價格和效能考量](../articles/sql-database/sql-database-elastic-pool-guidance.md)。
