部署指令碼在偵測到已有相容的虛擬環境存在時，將會略過在 Azure 上建立虛擬環境的作業。這可以大幅加速部署。pip 將會略過已安裝的封裝。

在某些情況下，您可能想要強制刪除該虛擬環境。如果您決定將虛擬環境納入為儲存機制的一部分，就會這麼做。您也有可能因為需要去除特定封裝或測試 requirements.txt 的變更，而想這麼做。

有幾個選項可用來在 Azure 上管理現有的虛擬環境：

### 選項 1：使用 FTP

使用 FTP 用戶端連接到伺服器，您將能夠刪除 env 資料夾。請注意，某些 FTP 用戶端 (例如網頁瀏覽器) 可能是唯讀的，而且不允許您刪除資料夾，所以您應確實使用 FTP 用戶端來執行該功能。FTP 主機名稱和使用者會在 Azure 入口網站上顯示於您網站的儀表板頁面上。

### 選項 2：切換執行階段

部署指令碼在 env 資料夾不符合所需的 Python 版本時會刪除該資料夾，下列替代方案將會利用此一特性。這將會有效刪除現有的環境，並建立新環境。

1. 切換至不同版本的 Python (透過 runtime.txt 或網站設定頁面)
1. git 推送某些變更 (忽略任何發生的 pip 安裝錯誤)
1. 切換回初始版本的 Python
1. 再次 git 推送某些變更

### 選項 3：自訂部署指令碼

如果您已自訂部署指令碼，您可以變更 deploy.cmd 中的程式碼，以強制它刪除 env 資料夾。
