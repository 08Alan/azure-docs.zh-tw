除了自動將用戶端連接至主要複本，接聽程式也可以用來將唯讀工作負載重新導向至次要複本。這可以改善整體解決方案的效能和延展性。如需詳細資訊，請參閱[使用 ReadIntent 路由搭配 Azure AlwaysOn 可用性群組接聽程式](http://go.microsoft.com/fwlink/?LinkId=522515)。

>[AZURE.NOTE] 如需 Azure 接聽程式的疑難排解秘訣，請參閱 AlwaysOn 支援小組[部落格](http://blogs.msdn.com/b/alwaysonpro/)中的 [Troubleshooting Availability Group Listener in Azure (排解 Azure 中可用性群組接聽程式的問題)](http://blogs.msdn.com/b/alwaysonpro/archive/2016/02/01/troubleshooting-availability-group-listener-in-azure.aspx)。

如需在 Azure 中使用 SQL Server 的其他資訊，請參閱 [Azure 虛擬機器上的 SQL Server](../articles/virtual-machines/virtual-machines-windows-classic-sql-overview.md)。
