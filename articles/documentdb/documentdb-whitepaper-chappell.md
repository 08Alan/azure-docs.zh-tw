# <a name="introducing-documentdb---a-nosql-database-for-microsoft-azure"></a>DocumentDB 簡介 - 適用於 Microsoft Azure 的 NoSQL 資料庫 (英文)
> [!NOTE]
> 本白皮書上次更新時間是在 2015 年，尚未更新來納入 DocumentDB 的更新，例如[資料全球發佈](documentdb-distribute-data-globally.md)和[分割的集合](documentdb-partition-data.md)。 如需 DocumentDB 目前的簡介，請參閱 [DocumentDB 簡介：NoSQL JSON 資料庫](documentdb-introduction.md)。
> 
> 

不確定 DocumentDB 是否適合您的應用程式或資料？ 想要深入了解 JSON 資料是如何在 Azure DocumentDB 中儲存及存取？ 如果是這樣，David Chappell 所著的本簡介白皮書是絕佳起點。 其中會討論如何能從無結構描述的 JSON 資料提供的彈性獲得優勢，讓您毫不費力地處理資料結構的變更。 

本白皮書還會逐步引導您執行下列動作：

* 使用 DocumentDB 資料模型將 JSON 資料儲存在文件中和資料庫內的集合。 
* 使用用戶端程式庫、RESTful 存取方法或 DocumentDB SQL，使用您的資料。 
* 從資料庫本身執行邏輯，方法是撰寫預存程序 (sproc)、觸發程序或使用者定義函數 (UDF)，並將它們儲存在集合中。
* 選擇一致性選項，以符合您的應用程式的可靠性和效能需求。
* 根據您的輸送量需求選取價格選項。
