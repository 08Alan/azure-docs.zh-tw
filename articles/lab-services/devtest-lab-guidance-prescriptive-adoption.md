# <a name="set-up-azure-devtest-labs-infrastructure-in-your-enterprise"></a>在企業中設定 Azure DevTest Labs 基礎結構
企業快速採用雲端的原因在於其具有靈活度、彈性和經濟展望上的優勢。 進入雲端的第一個步驟通常是移轉開發和測試工作負載。 移轉這些工作負載時有一組常見考量，如下： 

- 保護開發/測試資源
- 清楚地區隔開發/測試與生產環境
- 劃分 IT、應用程式和專案小組之間的資源
- 管理成本
- 允許自助設定開發/測試基礎結構，而不會危及安全性

## <a name="intended-audience"></a>目標對象
本文件適用於 IT 規劃人員、架構設計人員和管理員，他們負責建立和檢閱整體部署，以及監督作業做法。 因此，本文件會強調整個程序和建議的設計準則，以提升安全且穩定的開發環境，最終促使在組織內採用 Azure DevTest Labs。
