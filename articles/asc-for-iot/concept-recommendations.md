# <a name="security-recommendations"></a>安全性建議

> [!IMPORTANT]
> Iot 的 ASC 目前處於公開預覽狀態。
> 此預覽版本是在沒有服務等級協定的情況下提供，不建議用於生產工作負載。 可能不支援特定功能，或可能已經限制功能。 如需詳細資訊，請參閱 [Microsoft Azure 預覽版增補使用條款](https://azure.microsoft.com/support/legal/preview-supplemental-terms/)。

ASC iot 進行中的方案分析為基礎，提供下列建議時協助改善及保護您的裝置、 操作狀態和整體的 IoT 中樞環境所需。 


## <a name="device-recommendations"></a>裝置的建議

裝置建議提供深入解析及建議來改善裝置的安全性和行為。 

| 嚴重性 | 名稱                                                      | 資料來源 | 描述                                                                                                                                                                                           |
|----------|-----------------------------------------------------------|-------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 中   | 在裝置上的開啟連接埠                                      | 代理程式       | 在裝置上找到接聽的端點                                                                                                                                                          |
| 中   | 其中一個鏈結中的寬鬆的防火牆原則。 | 代理程式       | 允許 （輸入/輸出） 中找到的防火牆原則。 防火牆原則應該預設會拒絕所有流量，並定義規則以允許從裝置所需的通訊。                               |
| 中   | 找不到輸入鏈結中的寬鬆的防火牆規則     | 代理程式       | 其中包含各種不同的 IP 位址或連接埠的寬鬆模式找到在防火牆中的規則。                                                                                    |
| 中   | 找不到輸出鏈結中的寬鬆的防火牆規則    | 代理程式       | 其中包含各種不同的 IP 位址或連接埠的寬鬆模式找到在防火牆中的規則。                                                                                   |
| 中   | 操作系統基準驗證失敗           | 代理程式       | 裝置不符合[CIS Linux 基準測試](https://www.cisecurity.org/cis-benchmarks/)                                                                                                         |

### <a name="operational-recommendation"></a>操作的建議

操作的建議提供深入解析及建議來改善代理程式設定。

| 嚴重性 | 名稱                                    | 資料來源 | 描述                                                                       |
|----------|-----------------------------------------|-------------|-----------------------------------------------------------------------------------|
| 低      | 代理程式會傳送未運用的訊息          | 代理程式       | 10%或更多的安全性訊息已在過去 24 小時內小於 4 kb。  |
| 低      | 安全性並非最佳的對應項設定 | 代理程式       | 對應項的安全性組態不是最佳的。                                        |
| 低      | 安全性對應項的組態衝突    | 代理程式       | 安全性對應項組態中發現衝突。                           |


## <a name="iot-hub-recommendations"></a>IoT 中樞的建議

建議的警示提供深入解析與建議，以改善您的環境的安全性狀態的動作。  

| 嚴重性 | 名稱                                                     | 資料來源 | 描述                                                                                                                                                                                                             |
|----------|----------------------------------------------------------|-------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 高     | 多個裝置所使用的相同的驗證認證 | IoT 中樞     | 多個裝置會使用 IoT 中樞驗證認證。 這可能表示非法的裝置模擬合法的裝置。 重複認證使用會增加惡意執行者的裝置模擬的風險。 |
| 中   | 應該拒絕預設 IP 篩選器原則                  | IoT 中樞     | IP 篩選器設定應該有允許流量，而且也應該預設為定義的規則，預設為拒絕所有其他流量。                                                                                                     |
| 中   | IP 篩選器規則都包含大型的 IP 範圍                   | IoT 中樞     | 太大而允許的 IP 篩選器規則來源 IP 範圍。 過度寬鬆的規則可以公開您的 IoT 中樞，惡意的動作項目。                                                                                       |
| 低      | 啟用 IoT 中樞的診斷記錄                       | IoT 中樞     | 啟用記錄並保留記錄最長一年。 保留記錄檔，可讓您重新建立活動記錄來進行調查，當發生安全性事件或遭到入侵您的網路。                                       |
