# <a name="unified-alerts-in-azure-monitor"></a>Azure 監視器中的整合警示

## <a name="overview"></a>概觀

> [!NOTE]
>  新的整合警示體驗可讓您管理多個訂用帳戶中的警示，並引入目前僅供公開預覽的警示狀態和智慧群組。 關於這方面增強的體驗和啟用的程序，請參閱[本文的最後一節](#enhanced-unified-alerts-experience-public-preview)。


本文說明 Azure 監視器中的整合警示體驗。 Azure 監視器功能表的 [警示 (傳統)] 選項提供[以往的警示體驗](monitoring-overview-alerts.md)。 

## <a name="features-of-the-unified-alert-experience"></a>整合警示體驗的功能

與傳統體驗相比，整合體驗具有下列優點：

-   **更好的通知系統**：整合警示都使用[動作群組]( https://docs.microsoft.com/azure/monitoring-and-diagnostics/monitoring-action-groups)，這些是可在多個警示中重複使用的具名通知及動作群組。 
- **整合撰寫體驗** - 對於「Azure 監視器」、Log Analytics 和 Application Insights，您可以集中管理計量、記錄和活動記錄的警示和警示規則。 
- **檢視 Azure 入口網站中引發的 Log Analytics 警示** - 檢視 Log Analytics 發出的警示，以及 Azure 入口網站中的其他來源發出的其他警示。 先前這些警示是在個別的入口網站中。
- **引發的警示與警示規則之間的分隔** - 警示規則現在與警示相區隔。 警示規則會定義觸發警示的條件。 警示是警示規則引發的執行個體。
- **更好的工作流程** - 整合警示撰寫體驗會引導您完成設定警示規則的程序。
 
計量警示具有優於傳統計量警示的下列改善：

-   **改善延遲**：計量警示的執行頻率可以達到每分鐘一次。 傳統計量警示的執行頻率一律是每 5 分鐘一次。 記錄警示仍然有超過 1 分鐘的延遲，因為會花費時間內嵌記錄。 
-   **支援多維度計量**：您可以針對維度計量發出警示，以便監視特定的計量執行個體。
-   **更充分掌控計量條件**：您可以定義更多警示規則支援監視計量的最大值、最小值、平均值和總計值。
-   **可合併監視多個計量**：您可以使用單一規則來監視最多兩個計量。 若兩個計量在指定的期間內都超出其個別閾值，就會觸發警示。
-   **來自記錄的計量** (有限公開預覽)：現在可將進入 Log Analytics 的某些記錄資料擷取並轉換成「Azure 監視器」計量，然後針對這些計量發出警示，就像任何其他計量一樣。 


## <a name="alert-rules"></a>警示規則
整合警示體驗使用下列概念來分隔警示規則和警示，同時在不同的警示類型之間維持整合的編寫體驗。

| Item | 定義 |
|:---|:---|
| 警示規則 | 定義建立警示的條件。 由_目標資源_、_訊號_、_準則_和_邏輯_組成。 只有_已啟用_狀態的警示規則才有作用。
| 目標資源 | 定義可用於警示的特定資源和訊號。 目標可以是任何 Azure 資源。<br>範例：虛擬機器、儲存體帳戶、虛擬機器擴展集、Log Analytics 工作區、Application Insights 資源 |
| 訊號 | 目標資源所發出的資料來源。 支援的訊號類型包括*計量*、*活動記錄*、*Application Insights* 及*記錄*。 |
| 準則 | 目標資源上所套用_訊號_和_邏輯_的組合。<br>範例：百分比 CPU > 70%、伺服器回應時間 > 4 毫秒、記錄查詢的結果計數 > 100 等等。 |
| 邏輯 | 可檢查訊號是否在預期範圍/值內的使用者定義邏輯。 |
| 動作 | 引發警示時將執行的動作。 引發警示時，可能會進行多個動作。 這些警示支援動作群組。<br>例如，傳送郵件給某個電子郵件地址、呼叫 Webhook URL。 |
| 監視條件 | 指出後續是否已解決建立計量警示的條件。 計量警示規則會按照固定間隔進行特定計量的取樣。 如果符合警示規則的條件，將以 [已引發] 的條件建立新的警示。  再次進行計量的取樣時，如果仍然符合準則，則不會進行任何動作。  如果不符合條件，警示的條件會變更為 [已解決]。 下次符合條件時，將以 [已引發] 的條件建立另一個警示。 |

整合警示提供檢視和管理所有 Azure 警示的單一位置。 下列各小節說明整合體驗每個個別頁面的功能。

[警示] 概觀頁面會顯示所有引發之警示及已啟用之警示規則總計的彙總摘要。 變更訂用帳戶或篩選參數會更新彙總和引發警示的清單。

### <a name="alert-rules-management"></a>警示規則管理
[規則] 是可管理各個 Azure 訂用帳戶之所有警示規則的單一頁面。 它會列出所有警示規則，並可根據目標資源、資源群組、規則名稱或狀態來排序。 您也可以從這個頁面編輯、啟用或停用警示規則。

 ![警示規則](./media/monitoring-overview-unified-alerts/alerts-preview-rules.png)


## <a name="creating-an-alert-rule"></a>建立警示規則
不論監視服務或訊號類型為何，都能以一致的方式編寫警示。 所有引發的警示和相關的詳細資料都在單一頁面中提供。
 
您可以使用下列三個步驟建立新的警示規則：
1. 挑選警示的_目標_。
1. 從目標的可用訊號中選取_訊號_。
1. 從訊號指定套用於資料的_邏輯_。
 
這個簡化的編寫流程讓使用者不再需要先知道監視來源或支援的訊號，就能選取 Azure 資源。 可用訊號的清單會根據選取的目標資源自動篩選，並引導定義警示規則的邏輯。

您可參閱[使用 Azure 監視器來建立、檢視及管理警示](monitor-alerts-unified-usage.md)，深入了解如何建立警示規則。

警示可跨數個 Azure 監視服務使用。 如需如何和何時使用每個服務，請參閱[監視 Azure 應用程式和資源](./monitoring-overview.md)。 下表列出 Azure 提供的警示規則類型，以及整合警示體驗目前支援的警示。

| **監視來源** | **訊號類型**  | **說明** | 
|-------------|----------------|-------------|
| Azure 監視器 | 計量  | 也稱為[近乎即時計量警示](monitoring-near-real-time-metric-alerts.md)，這些警示支援以 1 分鐘一次的頻率評估計量條件，並允許多計量及多維度規則。 在 [Azure 入口網站中 Azure 服務的新版計量警示](monitoring-near-real-time-metric-alerts.md#metrics-and-dimensions-supported)，可取得支援之資源類型的清單。<br>新的警示體驗不支援[傳統計量警示](monitoring-overview-alerts.md)。 您可以在 Azure 入口網站中的 [警示 (傳統)] 底下找到它們。 傳統警示支援一些尚未移至新版警示的計量類型。 如需完整的清單，請參閱[支援的計量](https://docs.microsoft.com/azure/monitoring-and-diagnostics/monitoring-supported-metrics)。 |
| Log Analytics | 記錄檔  | 記錄搜尋查詢符合特定準則時，接收通知或執行自動化動作。 Log Analytics 中的警示會[複製到新的體驗](monitoring-alerts-extend.md)。 有[預覽版的以 *Log Analytics 記錄作為計量*](monitoring-alerts-extend-tool.md)可供使用。 此預覽版可讓您將一些類型的記錄轉換成計量，然後使用新的警示體驗來針對它們發出警示。 如果您有想要與原生「Azure 監視器」計量搭配使用的非 Azure 記錄，此預覽版會相當有用。 |
| 活動記錄檔 | 活動記錄檔 | 包含選取之目標建立的所有建立、更新和刪除動作。 |
| 服務健康情況 | 活動記錄檔  | 不支援整合警示。 請參閱[建立服務通知的活動記錄警示](monitoring-activity-log-alerts-on-service-notifications.md)。  |
| Application Insights | 記錄檔  | 包含具有應用程式效能詳細資料的記錄。 您可以使用分析查詢，定義要根據應用程式資料來採取動作的條件。 |
| Application Insights | 計量 | 不支援整合警示。 請參閱 [計量警示]。(../application-insights/app-insights-alerts.md) |
| Application Insights | Web 可用性測試 | 不支援整合警示。  請參閱 [Web 測試警示](../application-insights/app-insights-monitor-web-app-availability.md)。 可供任何經檢測可傳送資料給 Application Insights 的網站使用。 當網站的可用性或回應能力低於預期時收到通知。 |

## <a name="enhanced-unified-alerts-public-preview"></a>增強的整合警示 (公開預覽)
> [!NOTE]
>  本節中的功能即將推出。 這可能不會出現在入口網站的版本中。 

增強的整合警示體驗在 2018 年 6 月 1 日 Azure 監視器的公開預覽中推出。 此體驗是以 2018 年 3 月[整合警示](#overview) 的優點為基礎，且您可以管理和彙總個別警示，並修改警示狀態。 本節說明新功能以及如何瀏覽 Azure 入口網站中的新警示頁面。

### <a name="features-enhanced-unified-alerts"></a>增強整合警示的功能

新的體驗提供傳統整合體驗不提供的下列功能：

- **檢視訂用帳戶的警示** - 您現可在單一檢視中檢視和管理多個訂用帳戶的個別警示執行個體。
- **管理警示的狀態** - 警示現在有狀態指示是否已通過關閉的認可。
- **使用智慧群組組織警示** - 智慧群組會自動將相關的警示分組，因此您可以管理警示的群組，而非管理個別警示。

### <a name="enable-enhanced-unified-alerts"></a>啟用增強整合警示
按一下 [警示] 頁面最上方的橫幅，啟用新的整合警示體驗。 此程序會建立警示存放區，其中包含支援的服務在過去 30 天引發的警示。 啟用新的體驗時，您可以按一下橫幅來回切換新舊體驗。

> [!NOTE]
>  讓新體驗初次啟用時，可能需要幾分鐘的時間。

![橫幅](media/monitoring-overview-unified-alerts/opt-in-banner.png)

您啟用新體驗時，將註冊您可存取的所有訂用帳戶。 雖然整個訂用帳戶已啟用，但是只有選取新體驗的使用者才能夠檢視訂用帳戶。 可存取訂用帳戶的其他使用者必須個別啟用體驗。

啟用新的警示體驗時，不會影響您警示規則中的動作群組或通知設定。 這只會變更您在 Azure 入口網站中檢視和管理引發的警示執行個體所用的方式。

### <a name="smart-groups"></a>智慧群組
智慧群組可減少干擾，讓您專注於管理一個單位的相關警示，而不需要管理個別警示。 您可以檢視智慧群組的詳細資料，並設定與警示類似的狀態。 每個警示都是單一專屬智慧群組的成員。

智慧群組會自動使用機器學習加以建立，以便結合代表單一問題的相關警示。 建立警示時，演算法會根據歷史模式、屬性相似性和結構相似性等資訊，將警示加入至新的智慧群組或現有的智慧群組。 目前，演算法只會考慮訂用帳戶之內相同監視器服務的警示。 智慧群組可以透過此彙總減少高達 99% 的警示干擾。 您可以檢視智慧群組詳細資料頁面中的群組包含警示的原因。

智慧群組的名稱是其第一個警示的名稱。 您無法建立或重新命名智慧群組。


### <a name="alert-states"></a>警示狀態
增強整合警示導入警示狀態的概念。 您可以設定警示的狀態來指定警示在解決流程中的位置。  建立警示時，警示的狀態為 [新增]。 您認可並關閉警示時，可以變更狀態。 任何狀態變更都會儲存在警示的記錄中。

支援下列警示狀態：

| State | 說明 |
|:---|:---|
| 新增 | 已偵測到問題，而且尚未檢閱問題。 |
| 已認可 | 系統管理員已檢閱警示，且已開始處理。 |
| 關閉 | 已解決問題。 警示已關閉時，您可以重新開啟警示，將它變更為另一個狀態。 |

警示的狀態與監視器條件不同。 不再符合錯誤條件時，計量警示規則可將警示設定為 [已解決]的條件。 警示狀態是由使用者設定，而與監視器條件無關。 即使系統可能將監視器條件設定為已解決，在使用者變更警示狀態前，警示狀態不會變更。

#### <a name="changing-the-state-of-an-alert-or-smart-group"></a>變更警示或智慧群組的狀態
您可設定智慧群組的狀態，以變更個別警示的狀態或同時管理多個警示。

按一下警示之詳細資料檢視中的 [變更警示狀態] 變更警示狀態，或按一下其詳細資料檢視中的 [變更智慧群組狀態] 變更智慧群組的狀態。 您可以在清單檢視中選取多個項目，並按一下頁面頂端的 [狀態變更]，一次變更多個項目的狀態。 在這兩種情況下，從下拉式清單中選取新的狀態，並選擇提供註解。 如果您要變更單一項目，則您也可以選擇將相同的變更套用於智慧群組中的所有警示。

![變更狀態](media/monitoring-overview-unified-alerts/change-tate.png)

### <a name="alerts-page"></a>警示頁面
對於特定時間內建立的警示，預設的 [警示] 頁面提供警示的摘要。 這會顯示每個嚴重性的警示總計，且有資料行針對每個嚴重性顯示每個狀態的警示總數。 按一下任何嚴重性開啟依照該嚴重性篩選的 [所有警示][](#all-alerts-page) 頁面。

![警示頁面](media/monitoring-overview-unified-alerts/alerts-page.png)

您可以選取頁面頂端下拉式清單中的值來篩選這份檢視。

| 欄 | 說明 |
|:---|:---|
| 訂用帳戶 | 選取最多 5 個 Azure 訂用帳戶。 檢視僅會包含所選訂用帳戶中出現的警示。 |
| 資源群組 | 選取單一資源群組。 檢視僅會包含所選資源群組中具有目標的警示。 |
| 時間範圍 | 檢視僅會包含所選時間內引發的警示。 支援的值是過去 1 小時、過去 24 小時、過去 7 天和過去 30 天。 |

按一下 [警示] 頁面頂端的下列值將開啟另一個頁面。

| 值 | 說明 |
|:---|:---|
| 警示總計 | 符合所選準則的警示總數。 按一下此值將開啟沒有任何篩選的 [所有警示] 檢視。 |
| 智慧群組 | 從符合所選準則之警示所建立智慧群組的總數。 按一下此值將開啟 [所有警示] 檢視中的 [智慧群組] 清單。
| 警示規則總計 | 所選訂用帳戶和資源群組中的警示規則總數。 按一下此值將開啟在選取的訂用帳戶與資源群組上篩選的規則檢視。


### <a name="all-alerts-page"></a>[所有警示] 頁面 
[所有警示] 頁面可讓您檢視所選時間內建立的警示清單。 您可以檢視個別警示的清單或包含警示的智慧群組清單。 按一下頁面頂端的橫幅切換檢視。

![[所有警示] 頁面](media/monitoring-overview-unified-alerts/all-alerts-page.png)

您可以選取頁面頂端下拉式清單中的下列值來篩選這份檢視。

| 欄 | 說明 |
|:---|:---|
| 訂用帳戶 | 選取最多 5 個 Azure 訂用帳戶。 檢視僅會包含所選訂用帳戶中出現的警示。 |
| 資源群組 | 選取單一資源群組。 檢視僅會包含所選資源群組中具有目標的警示。 |
| 資源類型 | 選取一個或多個資源類型。 檢視僅會包含所選類型目標之具目標的警示。 指定資源群組之後，才可使用此資料行。 |
| 資源 | 選取資源。 檢視僅會包含使用該資源做為目標的警示。 指定資源類型之後，才可使用此資料行。 |
| 嚴重性 | 選取警示嚴重性，或選取 [所有] 以包含所有嚴重性的警示。 |
