# <a name="azure-cosmos-db-documentation-experiment"></a>Azure Cosmos DB 文件實驗

Azure Cosmos DB 是全域散發的多模型資料庫。 因為您可以藉由 Azure Cosmos DB 執行許多工作 (從建立圖表到建立文件資料庫，或是從 MongoDB、Cassandra 或「Azure 資料表」儲存體移轉資料)，所以組織文件會相當具挑戰性。 因此，我們希望有您協助。

我們正在測試 Azure Cosmos DB 目錄的組織和[登陸頁面](https://docs.microsoft.com/en-us/azure/cosmos-db/) (如下圖所示)，而希望取得您對這些所提議變更的意見反應。

## <a name="to-load-both-views"></a>同時載入兩個檢視

1. 若要並排開啟這兩個檢視，請從您的慣用網頁瀏覽器瀏覽至[登入頁面](https://docs.microsoft.com/en-us/azure/cosmos-db/)。 
2. 使用上述圖像來判斷您所看到的版本是哪一個 (原始或實驗性)。 
3. 開啟 InPrivate 或無痕瀏覽器視窗，並在新瀏覽器視窗中載入[登陸頁面](https://docs.microsoft.com/en-us/azure/cosmos-db/)，然後同樣使用上述圖像來判斷您所看到的版本是哪一個。 

   如果未同時顯示兩個檢視，請關閉後再重新開啟新的 InPrivate 或無痕視窗，然後重新載入[登陸頁面](https://docs.microsoft.com/en-us/azure/cosmos-db/)。 可能需要嘗試幾次，才能在個別的視窗中載入這兩個版本。 重新整理無痕視窗並不會載入另一個版本，您必須開啟新視窗。 

4. 您可以接著調整視窗大小來並排固定，或是在檢視之間來回切換以進行比較。 

5. 在您已有機會熟悉實驗性體驗的組織和版面配置之後，請按一下下方的 [實驗問卷] 按鈕來提供意見反應。 請在 2018 年 4 月 4 日午夜之前提交表單，以便我們檢閱和合併該意見反應。
