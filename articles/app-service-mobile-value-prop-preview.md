# <a name="getting-started"> </a>什麼是行動應用程式

Azure 應用程式服務是完全受管理的平台即服務 (PaaS) 產品，適用於專業開發人員，具有一組豐富的功能可用於 Web、行動和整合案例。 *Azure App Service* 中的  *Mobile Apps* 為企業開發人員與系統整合業者提供了具高度擴充性、供全域使用的行動應用程式開發平台，附有一組內容豐富、適用於行動應用程式的功能集。

![Mobile Apps](./media/app-service-mobile-value-prop-preview/overview.png)

# 為何要使用行動應用程式？
 *Azure App Service* 中的*行動應用程式*為企業開發人員與系統整合業者提供了具高度擴充性、供全域使用的行動應用程式開發平台，附有一組內容豐富、適用於行動應用程式的功能集。透過行動應用程式，您將可：

- **建置原生和跨平台應用程式** - 無論您要建置原生 iOS、Android 和 Windows 應用程式還是跨平台的 Xamarin 或 Cordova (Phonegap) 應用程式，您可以利用採用原生 SDK 的應用程式服務。
- **連接到您的企業系統** - 透過行動應用程式，您將可在短短幾分鐘內新增公司登入，並連接到企業內部部署或雲端資源。
- **輕鬆連接到 SaaS API** - 藉由超過 40 個的 SaaS API 連接器，您可以輕鬆地整合您的應用程式與企業目前使用的 SaaS API。想要同時更新 CRM 和計費系統中的帳戶狀態嗎？行動應用程式提供了方便取用的企業 SaaS API。
- **透過同步處理建置可離線使用的應用程式** - 建置可離線運作的應用程式以提高行動工作人員的生產力，並且在您任何的企業資料來源或 SaaS API 使用連線時，使用行動應用程式在背景中同步處理資料。
- **在短短數秒內推播通知給數百萬人** - 根據客戶的個人需求，使其在任何裝置上適時接收即時的推播通知。

## 行動應用程式概念
- **單一登入** - 從持續增長的身分識別提供者清單中選取 (包括 Azure Active Directory、Facebook、Google、Twitter 和 Microsoft 帳戶)，並運用行動應用程式，在幾分鐘內將驗證新增至您的應用程式。
- **離線同步** - 行動應用程式可讓您輕鬆建置健全、反應迅速的應用程式，讓員工可在連線無法使用時離線工作，並在裝置恢復連線時與您的企業後端系統進行同步處理。離線同步處理功能在所有平台上皆受到支援，並且可與任何資料來源 (包括 SQL、資料表儲存體、Mongo 或文件 DB) 和任何 SaaS API (包括 Office 365、Salesforce、Dynamics) 或內部部署資料庫搭配運作。 
- **推播通知** - 行動應用程式提供可大幅擴充的行動推播通知引擎「通知中心」，能夠在數秒內將數以百萬計的個人化推播通知傳送至使用 iOS、Android、Windows 或 Kindle 裝置之目標對象的動態區段。您可以輕鬆地將通知中心連結至任何現有的應用程式後端，無論該後端是在內部部署還是雲端中受到主控。
- **自動調整** - 應用程式服務可讓您快速地向上或向外擴充，以處理任何內送的客戶負載。您可以手動選取 VM 的數量和大小，或設定自動調整，來調整您的行動應用程式後端或排程。
