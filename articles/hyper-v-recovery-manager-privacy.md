# Azure Site Recovery 的隱私權資訊

	<td><p><b>註冊</b></p></td>
	<td><p>向服務註冊伺服器，如此可以保護虛擬機器</p></td>
	<td><p>註冊服務之後，服務會收集處理程序並傳輸這項資訊：</p>
		<ul>
			<li>來自 VMM 伺服器的管理憑證資訊，該 VMM 伺服器指定為使用其服務名稱提供災害復原。</li>
			<li>VMM 伺服器上的虛擬機器雲端名稱</li>
		</ul>
</td>
	<td><p>服務會使用上述資訊，如下所示：</p>
		<ul>
			<li>管理憑證-這是用來協助識別及驗證已註冊的 VMM 伺服器，以便存取服務。服務會使用憑證的公開金鑰部分來保護權杖，只有已註冊的 VMM 伺服器可以獲得其存取權。伺服器必須使用此權杖獲得服務功能的存取權。</li>
			<li>VMM 伺服器的名稱-VMM 伺服器名稱必須識別雲端所在的適當 VMM 伺服器並與其進行通訊。 
			</li>
			<li>VMM 伺服器的雲端名稱-使用以下所述之服務雲端配對/取消配對功能時，雲端名稱為必要項目。當您決定配對主要資料中心的雲端與另一個在復原資料中心的雲端時，復原資料中心的所有雲端名稱都會出現。</li>
		</ul>
</td>
	<td><p>這項資訊是服務註冊程序中不可或缺的一部分，因為它可幫助您和服務識別您要提供 Azure Site Recovery 保護的 VMM 伺服器，並且識別正確註冊的 VMM 伺服器。如果您不想將此資訊傳送給服務，請勿使用此服務。如果您已註冊您的伺服器，且稍後想要取消註冊，您可以從服務入口網站 (也就是 Azure 入口網站) 刪除 VMM 伺服器資訊，即可完成動作。</p></td>
</tr>

<tr>
	<td><p><b>啟用 Azure Site Recovery 保護</b></p></td>
	<td><p>安裝在 VMM 伺服器上的 Azure Site Recovery 提供者是用來和服務通訊的管道。提供者是裝載在 VMM 程序中的動態連結程式庫 (DLL)。安裝提供者之後，"Datacenter Recovery" 功能會在 VMM 系統管理員主控台中啟用。雲端中所有新的或現有的虛擬機器都可以啟用稱為 "Datacenter Recovery" 的屬性以協助保護虛擬機器。一旦設定此屬性之後，提供者會將虛擬機器的名稱和識別碼傳送至服務。虛擬保護是由 Windows Server 2012 或 Windows Server 2012 R2 Hyper-V 複寫技術啟用。虛擬機器資料會從一個 Hyper-V 主機複寫至另一個 (通常位於不同的「復原」資料中心)。</p></td>
	<td><p>服務會收集、處理和傳輸虛擬機器的中繼資料，包括名稱、識別碼、虛擬網路和所屬雲端。</p>
	</td>
	<td><p>服務會使用上述資訊，在您的服務入口網站填入虛擬機器資訊。</p>
	</td>
	<td><p>這是服務不可或缺的一部分，而且無法關閉。如果您不想將此資訊傳送給服務，請勿啟用任何虛擬機器的 Azure Site Recovery 保護。請注意，所有由提供者傳送給服務的資料都是透過 HTTPS 傳送。</p></td>
</tr>
<tr>
	<td><p><b>復原計畫</b></p></td>
	<td><p>此功能可協助您建置「復原」資料中心的協調計畫。您可以定義虛擬機器或虛擬機器群組應在復原站台啟動的順序。您也可以在每個虛擬機器復原時指定任何要執行的自動化指令碼，或任何要採取的手動動作。容錯移轉 (下一節所述) 通常會在協調復原的復原計畫層級觸發。</p></td>
	<td><p>服務會收集處理程序，並傳輸這項資訊做為復原計畫功能的一部分：</p>
		<ul>
			<li>復原計畫，包含虛擬機器中繼資料</li>
			<li>自動化指令碼或手動動作備註的中繼資料。</li>
		</ul>
	</td>
	<td><p>上述中繼資料可用來在您的服務入口網站建置復原計畫。</p>
	</td>
	<td><p>這是服務不可或缺的一部分，而且無法關閉。如果您不想將此資訊傳送給服務，請勿在此服務中建置復原計畫。</p></td>
</tr>
<tr>
	<td><p><b>網路對應</b></p></td>
	<td><p>這項功能可讓您將網路資訊從主要資料中心對應至復原資料中心。在復原站台上復原虛擬機器時，此對應可協助建立其網路連線。</p></td>
	<td><p>服務的網路對應功能之一是收集、處理和傳輸每個站台 (主要與資料中心) 之邏輯網路的中繼資料。</p>
	</td>
	<td><p>服務會使用中繼資料來填入服務入口網站，您可以在這個入口網站對應網路資訊。</p>
	</td>
	<td><p>這是服務不可或缺的一部分，而且無法關閉。如果您不想將此資訊傳送給服務，請勿使用網路對應功能。</p></td>
</tr>

		</ul>
