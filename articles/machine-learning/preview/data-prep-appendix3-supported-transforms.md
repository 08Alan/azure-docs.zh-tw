# <a name="supported-transforms-for-this-release"></a>此版本支援的轉換
以下列出並概述此版本的可用轉換。 

此版本支援的轉換如下所述。

## <a name="column-selection"></a>資料行選擇 
以下列出的許多轉換會在單一或多個資料行上運作。 若要選取多個資料行，請按住 Ctrl 同時按一下每個資料行，或按住 Shift 同時按一下資料行範圍。

## <a name="transforms-from-main-menu-andor-grid-header"></a>從主功能表和/或格線標頭轉換 
可從主功能表上的 [轉換] 選項存取轉換。 也可以用滑鼠右鍵按一下資料格線中的資料行名稱來選取轉換。 如果選取多個資料行，用滑鼠右鍵按一下任一資料行可提供轉換功能表。

用滑鼠右鍵按一下功能表，只會提供選取之資料類型的有效轉換，主功能表提供所有轉換，但是與所選取資料行不相關的轉換會呈現灰色。

透過在資料格中按一下滑鼠右件，可以獲得一小部分內容相關的轉換。 這些轉換包括複製、取代和篩選，這些轉換是資料類型感知，因此數字資料行的選項與字串資料行的選項不同。

## <a name="derive-column-by-example"></a>衍生資料行實例化
此轉換可建立新的資料行作為一或多個現有資料行的衍生。 它會藉由查看輸入 (選取的) 資料行與指定範例，以決定新資料行中所需的輸出。 

若要使用此做法，請選取一或多個資料行。 新增 (空白的) 衍生資料行實例化。 輸入您想要在衍生的資料行 (假設該資料行衍生自其他資料行) 中看見的範例，而且「實例化」技術會嘗試在資料行中填入所有其他資料格。 

對於複雜的範例，可能需要提供一個以上的範例，請選取其他資料格並輸入其他範例。

「實例化」技術使用選取的資料行嘗試判斷範例的意義。 如果叫用此轉換時，未選取任何資料行，則會使用目前資料列的所有資料格。 僅選取需要的資料行會產生更精確的結果。

您可以先選取資料行，然後再叫用轉換。 一旦啟動轉換編輯器，可以透過每個資料行頂端的核取方塊來查看選取哪些資料行作為輸入。 可以使用資料行標頭中的核取方塊來新增和移除選擇的資料行。

如需**衍生資料行實例化**轉換的詳細說明以及更多範例，請參閱：[衍生資料行實例化參考](data-prep-derive-column-by-example.md)  

## <a name="split-column-by-example"></a>分割資料行實例化
採用現有資料行並使用「實例化」引擎嘗試將資料行分割成 n 個其他資料行。 可以在後續產生的資料行上執行自動分割。

如需**衍生資料行實例化**轉換的詳細說明以及更多範例，請參閱：[分割資料行實例化參考](data-prep-split-column-by-example.md)

## <a name="expand-json"></a>展開 JSON

**展開 JSON**轉換可讓使用者藉由展開包含有效 JSON 文字的資料行，以新增多個資料行。

如需**展開 JSON** 轉換的詳細說明以及更多範例，請參閱：[展開 JSON 參考](data-prep-expand-json.md)


## <a name="combine-columns-by-example"></a>合併資料行實例化

可讓使用者藉由合併多個資料行的值以新增資料行。 

如需**合併資料行實例化**轉換的詳細說明以及更多範例，請參閱：[合併資料行實例化參考](data-prep-combine-columns-by-example.md)


## <a name="duplicate-column"></a>複製資料行
此轉換可完全複製資料行，並提供衍生自原始資料行名稱的新名稱。 此轉換支援一或多個資料行。

## <a name="text-clustering"></a>文字叢集 
此轉換旨在採用應該相同卻不一致的值，並將它們分組在一起。  

執行此轉換時，會針對單一資料行中的值以分析相似性。 然後將它們分組到叢集。 對於每個叢集而言，都有一個標準值，該值會取代叢集中所有執行個體，和執行個體值。 可以移除完整的叢集，可以編輯標準值。 可從指定的叢集移除執行個體。 此外，您可以變更用於將執行個體分組至叢集的相似性分數閾值篩選條件。

根據預設，這項轉換會將所有叢集執行個體值取代為該叢集的標準值，建立新的資料行以包含新值。 也可以將每個執行個體的相似性分數新增至新資料行 (可以命名該資料列)，以便稍後在資料流程中使用。

## <a name="replace-values"></a>取代值
此轉換允許一個字串被另一個字串取代，來源字串可以是部分字串或完整資料格。 取代可以套用至單一或多個資料行。 搜尋字串支援搜尋特殊字元以及一般字元。 

## <a name="replace-na-values"></a>取代 NA 值
可讓單一值取代各種不同形式的 NA (N/A、NA、null、NaN 等) 或空白字串，使其一致。 此轉換支援一或多個資料行。 只有在已選取資料行時，才會列出此轉換；未選取資料行時，主要轉換功能表上不會出現該轉換。

## <a name="replace-missing-values"></a>取代遺漏值
執行以單一值取代遺漏值。 此轉換支援一或多個資料行。 只有在已選取資料行時，才會列出此轉換；未選取資料行時，主要轉換功能表上不會出現該轉換。

## <a name="replace-error-values"></a>取代錯誤值
執行以單一值取代錯誤值。 此轉換支援一或多個資料行。 只有在已選取資料行時，才會列出此轉換；未選取資料行時，主要轉換功能表上不會出現該轉換。

## <a name="trim-string"></a>修剪字串
將開頭和結尾的「空白字元」(包含空格、定位字元等) 從一或多個資料行中移除。

## <a name="adjust-precision"></a>調整有效位數
允許設定數值資料行的小數位數

## <a name="rename-column"></a>重新命名資料行
變更資料行名稱，也可以按一下資料行的名稱，以在資料行標頭中內嵌叫用此轉換。

## <a name="remove-column"></a>移除資料行
移除選取的資料行，此轉換可在單一或多個資料行上產生效用。 

## <a name="keep-column"></a>保留資料行
僅保留選取的資料行，此轉換可在單一或多個資料行上產生效用。

## <a name="handle-path-column"></a>處理路徑資料行
在匯入檔案期間，精靈會將路徑資料行自動新增至資料集。 它包含形成資料集路徑的完整檔案名稱。 這項轉換可新增資料行或將該額外資料行從資料集移除。

## <a name="convert-field-type-to-numeric"></a>將欄位類型轉換成數值
將資料行類型變更為數值。 如果有非整數資料，請指定分隔符號。 根據預設，此轉換不會提示分隔符號，請使用 [編輯] 功能表項目以叫用編輯器。 此轉換適用於單一或多個資料行。

## <a name="convert-field-type-to-date"></a>將欄位類型轉換成日期
將資料行類型變更為日期。 會使用預設日期/時間格式，但可能會使用 `strftime` 指示詞覆寫。 也可以在固定日期添加時間值。

根據預設，此轉換不會提示格式，請使用產生之步驟的 [編輯] 功能表項目以叫用編輯器。 此轉換適用於單一或多個資料行。

只有 9-22-1677 和 4-11-2262 之間的日期可以轉換為日期類型。

## <a name="convert-field-type-to-boolean"></a>將欄位類型轉換成布林值
將資料行類型變更為 true/false。 此轉換支援將多個值對應至 true 或 false，可以編輯這些對應。 它也支援常數對應至 true/false 對應表中不存在的值。 此轉換適用於單一或多個資料行。

## <a name="convert-field-type-to-string"></a>將欄位類型轉換成字串
將資料行類型變更為字串。 此轉換適用於單一或多個資料行。

## <a name="convert-unix-timestamp-to-datetime"></a>將 Unix 時間戳記轉換為日期時間
即使在資料中表示為字串，並將其正確轉換為資料準備中的日期類型，此轉換也了解 Unix 時間戳記格式。

## <a name="filter"></a>Filter
篩選轉換支援根據一或多個資料行的值篩選資料列，篩選條件取決於每個資料行的資料類型；因此，可以透過「包含」或「不包含」篩選字串資料行，但不適用於數值。 但是，「大於」、「小於」條件適用於數值資料行，但不適用於字串資料行。

從主功能表或用滑鼠右鍵按一下資料行標頭以叫用篩選轉換的情況下，可使用選項將失敗的資料列派生至另一個資料流程。 因此，主要資料流程會繼續篩選**加入**資料列，並建立新的資料流程，其中包含篩選**移除**的所有資料列。

## <a name="use-first-row-as-headers"></a>使用第一個資料列作為標題
這個轉換會將資料集的第一個資料列升級為資料行名稱，如果某些資料行沒有第一個資料列中的資料，則會自動產生名稱。 使用此轉換表示最早從第 2 個資料列開始匯入資料。 根據「跳過資料列」設定，可能會在資料集更下層的資料列進行匯入。 它也可以用於移除提升功能，並且僅具備自動產生的名稱。

## <a name="join"></a>Join
此轉換用來聯結兩個資料流程。 它可根據預期結果來選擇聯結的輸出，例如聯結的成功資料列、聯結的「左側」失敗資料列，或聯結的「右側」失敗資料列。

「聯結」精靈會從單一資料流程中啟動，並選取該資料流程作為聯結的一端，接著會提示使用者針對聯結的另一端輸入另一個資料流程。 一旦選取兩個流程，精靈需要選取待聯結之每一端聯結的單一資料行。 如果聯結需要多個資料行，請先建立衍生資料行，然後再啟動精靈，以建立僅用於聯結的新 (串連) 資料行。 精靈會嘗試建立聯結索引鍵，如果可能，將自動產生衍生的資料行。

完成聯結後，會顯示聯結的 Sankey 圖表檢視，彼此相對的線條寬度代表透過聯接流程部分移動的資料列數。 右側面板可讓您選擇成功的資料列，左側失敗、右側非失敗，或可僅選擇一個分支。

## <a name="append-rows"></a>附加資料列
這項轉換會將其他資料流程的資料附加至目前的資料流程，它會按照位置對應資料列，以在最後新增資料列。

## <a name="append-columns"></a>附加資料行
這項轉換會將其他資料流程的新資料行附加至目前的資料流程，它會將新資料行新增至右側，但不會嘗試將資料排列在資料列中。

## <a name="summarize"></a>總結
選取資料行並針對唯一值的組合計算彙總。

支援的彙總：COUNT、SUM、MIN、MAX、MEAN、VARIANCE、STANDARD DEVIATION。 特定資料行的彙總清單會篩選為僅適用於該資料類型的清單。 不適用的彙總會呈現灰色。例如，無法計算字串資料行的平均值，但可以計算最小值和最大值。

類似於 ANSI-SQL GROUP BY

一旦編輯器為可用狀態，會從資料行標頭向上拖曳至左上方的面板，這裡會顯示要彙總的資料行。 此面板為階層式，因此可進行巢狀彙總。 右上方的編輯器面板可用於選取要套用至資料行的彙總。 單一資料行可以彙總一或多次。 選擇至少一個彙總之後，右下方的格線會預覽其彙總表單中的資料。 

## <a name="remove-duplicates"></a>移除重複項目
這項轉換會移除有重複值的整個資料列。 重複值是根據叫用轉換時選取的一或多個資料行來定義，或使用編輯器來定義。 如果未選擇任何資料行，則只會移除所有資料行值相同的資料列。

## <a name="sort"></a>排序
此轉換會排序資料，可依單一資料行或多個資料行完成，每個資料行可遞增 (預設) 或遞減排序 (可從編輯器中變更)。

類似於 ANSI-SQL ORDER BY

## <a name="output-transforms"></a>輸出轉換
下列轉換會輸出資料。 單一流程中可能會有多個寫入區塊，以便寫出不同點的資料。

### <a name="write-to-csv"></a>寫入至 CSV
這項轉換會從資料流程的目前點寫出 CSV 表單中的資料。  它允許控制位置 (本機或遠端) 以及檔案周圍的各種設定。

### <a name="write-to-parquet"></a>寫入至 Parquet
這項轉換會從資料流程中的目前點寫出 Parquet 表單中的資料。 它允許控制位置 (本機或遠端) 以及檔案周圍的各種設定。
