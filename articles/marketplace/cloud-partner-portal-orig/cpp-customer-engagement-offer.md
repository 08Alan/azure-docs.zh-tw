# <a name="dynamics-365-for-customer-engagement-offer"></a>Dynamics 365 for Customer Engagement 供應項目

<table> <tr> <td>本節說明如何將 Dynamics 365 for Customer Engagement 解決方案發佈到 Microsoft <a href="https://appsource.microsoft.com">AppSource Marketplace</a>。 Dynamics 365 for Customer Engagement 的所有應用程式 (Sales、Service、Project Service 及 Field Service) 都必須經過我們的認證程序並支援試用體驗。 認證程序會檢查您的解決方案以找出最佳做法。 試用體驗可讓使用者將解決方案部署至即時 Dynamics 365 環境。 </td> <td><img src="./media/dynce-icon1.png"  alt="Dynamics 365 Customer Engagement icon" /></td> </tr> </table>


## <a name="publishing-overview"></a>發佈概觀

[在 Microsoft AppSource (組建 2018) 上提供吸引人的體驗](https://www.youtube.com/watch?v=WWBFuNHC0J4)這部影片提供 AppSource Marketplace 的概觀，以及此生態系統的優點。 接著，還示範自訂的 Dynamics 365 for Customer Engagement 應用程式範例、說明提供「試用產品」試用的優點，以及如何發佈到 AppSource。

> [!Video https://channel9.msdn.com/events/Build/2018/THR1202/player]


## <a name="publishing-process"></a>發佈程序

本節中的後續文章將藉由下列步驟，引導您完成建立和發佈 Dynamics 365 for Customer Engagement 解決方案的程序：

1. 在「Microsoft 合作夥伴網路」(MPN) 上[註冊](https://partners.microsoft.com/PartnerProgram/simplifiedenrollment.aspx) (如果您尚未這麼做)。
2. [建立 Dynamics 365 解決方案](./dynamics-365-solution-preparation.md)。
3. [建立 AppSource 套件](./appsource-package-preparation.md)及相關行銷內容。
4.  將您的 AppSource 套件上傳到 Azure 儲存體，並[產生 SAS 金鑰](./store-your-appsource-package-to-azure-storage-and-generate-sas-key.md)。
5. [建立相關供應項目](./setup-your-application-in-the-publishing-portal.md)。
6. [發佈供應項目](./request-to-publish-the-app.md)以讓它上架。
