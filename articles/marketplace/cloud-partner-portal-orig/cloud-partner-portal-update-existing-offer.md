<a name="update-an-existing-offer-for-azure-marketplace"></a>更新 Azure Marketplace 的現有供應項目
==============================================

供應項目上線後，您可以對它進行各種更新。   [Cloud Partner 入口網站](https://cloudpartner.azure.com/)有數種功能可協助您編輯供應項目以確定變更是正確的，包括：

-  新增新的虛擬機器 (VM) 映像版本到現有的 SKU
-  變更可取得 SKU 的區域
-  新增 SKU
-  針對供應項目或 SKU 更新 Marketplace 中繼資料 
-  更新預付型方案供應項目的定價
-  比較功能
-  功能歷程記錄

修改供應項目或 SKU 之後，必須重新發行才能反映變更。
此文章會引導您完成更新 VM 供應項目的各種層面。


<a name="unpermitted-changes-to-vm-offersku"></a>不允許對 VM 供應項目/SKU 進行的變更
-----------------------------------

供應項目在 Azure Marketplace 中上線後，便無法修改 VM 供應項目或 SKU 的某些屬性。

-  供應項目的供應項目識別碼和發行者識別碼。
-  現有 SKU 的 SKU 識別碼。
-  現有 SKU 的資料磁碟計數。
-  現有 SKU 的計費/授權模式變更。


<a name="updating-the-vm-image-version-for-a-sku"></a>更新 SKU 的 VM 映像版本
---------------------------------------

可能已使用其他功能、安全性修補程式等更新供應項目之 SKU 的 VM 映像。在這類情況下，您可以更新您的 SKU 所參考的 VM 映像。 使用下列步驟來更新 VM 映像。 

1.  登入 [Cloud Partner 入口網站](https://cloudpartner.azure.com/)。
2.  在 [所有供應項目] 下，尋找您要更新的供應項目。
3.  在 [SKU] 表單下，按一下您要更新其 VM 映像的 SKU。
4.  在 [磁碟版本] 下，按一下 [+新增磁碟版本] 以新增 VM 映像。
5.  提供新的 VM 映像**磁碟版本**。 磁碟版本必須遵循[語意版本](http://semver.org/)格式。 版本格式應該是 X.Y.Z，其中 X、Y 和 Z 是整數。 請確定您提供的新版本大於先前的版本，否則新版本號碼在重新發佈時不會顯示在 Azure 入口網站或 Azure Marketplace 中。
6.  針對 [OS VHD URL] ，輸入為作業系統 VHD 建立的共用存取簽章 (SAS) URI。 請注意，無法變更不同 SKU 版本之間的資料磁碟計數。 如果先前的版本已設定資料磁碟，這個新版本也必須設定資料磁碟。
7.  按一下 [發佈] 開始進行發佈工作流程，以便讓新的 VM 版本在 Azure Marketplace 和 Azure 入口網站上線。


<a name="changing-regions-a-sku-is-available-in"></a>變更可取得 SKU 的區域
--------------------------------------

經過一段時間，您可能想要在更多區域提供您的供應項目/SKU。
或者，您可能想要在特定區域停止支援供應項目/SKU。
若要實作這些變更，請依照下列步驟執行。

1.  登入 [Cloud Partner 入口網站](https://cloudpartner.azure.com/)。
2.  在 [所有供應項目] 之下，尋找您要更新的供應項目。
3.  在 [SKU] 表單之下，按一下您要更新其區域可用性的 SKU。
4.  按一下 [國家/區域可用性] 欄位之下的 [選取國家 (地區)] 按鈕。
5.  在區域可用性快顯視窗中，新增/移除您想要提供此 SKU 的區域。
6.  按一下 [發佈] 開始進行發佈工作流程，以更新您的 SKU 區域可用性。

如果在新的區域中提供 SKU，您將可透過 [匯出價格資料] 功能來指定該特定地區的價格。 如果您加回之前曾經可用的區域，您將無法更新其價格，因為不允許變更價格。


<a name="adding-a-new-sku"></a>新增 SKU
----------------

依照下面的步驟讓新的 SKU 可用於現有的供應項目。

1.  登入 [Cloud Partner 入口網站](https://cloudpartner.azure.com/)。
2.  在 [所有供應項目] 之下，尋找您要更新的供應項目。
3.  在 [SKU] 表單之下，按一下 [新增 SKU] 並在快顯視窗中提供 [SKU 識別碼]。
4.  依照[將虛擬機器發行到 Azure Marketplace](./cloud-partner-portal-publish-virtual-machine.md) 中詳述的剩餘步驟執行。
5.  按一下 [發佈] 開始進行發佈工作流程，讓新的 SKU 上線。


<a name="updating-offer-marketplace-metadata"></a>更新供應項目 Marketplace 中繼資料。
------------------------------------

在某些情況下，您可能需要更新與您的供應項目相關聯的 Marketplace 中繼資料，例如更新您的公司標誌等。使用下列步驟來更新您供應項目的中繼資料。

1.  登入 [Cloud Partner 入口網站](https://cloudpartner.azure.com/)。
2.  在 [所有供應項目] 之下，尋找您要更新的供應項目。
3.  移至 [Marketplace] 表單並依照這裡的指示進行任何變更。
4.  按一下 [發佈] 開始進行發佈工作流程，讓您的變更生效。


<a name="updating-pricing-on-published-offers"></a>更新已發行之供應項目的定價
------------------------------------

一旦發行您的預付型方案供應項目，您就無法提高現有 SKU 的價格，但您可以在相同的供應項目下建立新的 SKU 並刪除舊 SKU，然後重新發行您的供應項目。 我們也允許您已發行之供應項目的價格。 降低您的供應項目價格：

1.  按一下您要降低價格的 SKU。
2.  若您已在 1x1 GUI 中設定定價，您可以直接在 UI 中變更定價。 若您透過匯入/匯出試算表設定定價，您只能透過匯入/匯出功能來降低價格。
3.  按一下 [檔案]。
4.  重新發行您的供應項目並讓它上架。

一旦供應項目在網站上完全上架，其定價就會顯示給新客戶，而且所有新客戶都只需要支付新的降價後價格。

針對現有的客戶，降低後的價格可追溯到降價開始生效時的帳單週期開始時間。
若客戶在降價時已被收取費用，他們將在下一個帳單週期收到退款單，以獲得降價後的價格補償。


<a name="simplified-currency-pricing"></a>簡化的貨幣定價

新的區段將接受以這些新貨幣表示的定價。 一旦所有客戶都移轉到這些新的結算貨幣之後，原始定價區段將不再使用，而且只會留下 [簡化的貨幣定價] 區段。

<a name="compare-feature"></a>比較功能
---------------

當您變更已發行的供應項目時，您可以使用「比較」功能來稽核已進行的變更。 使用比較功能：

1.  在編輯程序中的任何時間點，您都可以按一下您供應項目的 [比較] 按鈕。

2.  檢視行銷資產與中繼資料的並存版本。


<a name="history-of-publishing-actions"></a>發行動作的歷程記錄
-----------------------------

若要檢視任何歷史發行活動，請按一下 Cloud Partner 入口網站左邊瀏覽區中的 [歷程記錄] 索引標籤。 您可以在該處檢視在您的 Azure Marketplace 供應項目生命週期期間發生的動作 (含時間戳記)。
