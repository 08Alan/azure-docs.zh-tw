# <a name="azure-saas-application-offer"></a>Azure SaaS 應用程式供應項目

<table> <tr> <td>本節說明如何將軟體即服務 (SaaS) 應用程式發佈至 Microsoft <a href="https://azuremarketplace.microsoft.com/marketplace/">Azure Marketplace</a> 或 <a href="https://appsource.microsoft.com">Microsoft AppSource</a>。 當您的解決方案部署在您自己的 Azure 訂用帳戶中，且客戶將透過您所設計和管理的介面登入以測試應用程式時，請使用 SaaS 應用程式。 使用 <a href="https://azure.microsoft.com/services/active-directory">Azure Active Directory</a> (AAD) 可充分利用您現有的試用環境。 </td> <td><img src="./media/cloud-partner-portal-publish-web-app/saasapp-icon1.png"  alt="Azure SaaS application icon" /></td> </tr> </table>


## <a name="publishing-benefits"></a>發佈的優點

在 Microsoft Marketplace 上列出您的應用程式的部分優點包括：

- 在 Office 365 和 Dynamics 365 達到 100 萬個 Azure Active Directory 使用者。
- 擴充您的銷售團隊：覆蓋全球的商務使用者，並獲得吸引終端使用者、協助產生潛在客戶，並啟動跨行業新客戶對話的銷售管道。

- 取得可操作的深入解析：我們將會分享您的應用程式在 AppSource 上的表現，哪些部分效果良好，以及如何進一步提升銷售程序的深入見解。


## <a name="publishing-process"></a>發佈程序

若要在 Microsoft Azure Marketplace 或 Microsoft AppSource 上建立並發佈您的應用程式，本節將會引導您完成下列步驟：

1. 確認您符合先決條件，包括夥伴註冊：[開發人員中心註冊](https://dev.windows.com/en-us/registration?accountProgram=Azure)*或* [Microsoft 合作夥伴網路註冊](https://partner.microsoft.com/)。
2.  定義您的 Marketplace 供應項目和方案。
3.  提供行銷內容和定價。
4.  在 [Cloud Partner 入口網站](https://cloudpartner.azure.com)中測試並認證供應項目。
5.  讓您的供應項目上線。
