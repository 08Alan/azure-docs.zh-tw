# <a name="managing-azure-marketplace-and-appsource-publisher-profile"></a>管理 ‘Azure Marketplace’ 和 ‘AppSource’ 發行者設定檔

本文件逐步解說如何在您已註冊的發行者設定檔中檢視及管理使用者。 

此時，您已完成成為 Azure Marketplace 和 AppSource 發行者的步驟。您的合作夥伴要求獲得核准後，接著會在 **[Cloud Partner 入口網站](https://cloudpartner.azure.com/)**上註冊發行者設定檔。您的發行者設定檔將會套用到從合作夥伴註冊期間所用帳戶發佈的所有供應項目和 SKU。  

如果您尚未將貴公司註冊為雲端合作夥伴，請[按一下這裡](./cloud-partner-portal-getting-started-with-the-cloud-partner-portal.md)。


**發行者設定檔**︰您的發行者設定檔會區分 Marketplace 和 AppSource 上的貴公司。 它是由您的發行者識別碼、顯示名稱和擁有者電子郵件所組成。 管理完善的設定檔會增加您的能見度，並協助 Marketplace 使用者輕鬆地找出並選取適當的供應項目。

> [!NOTE]
>  
> 您一旦發佈第一個供應項目，便無法編輯您的發行者識別碼和已註冊的擁有者電子郵件。 不過，您可以檢視您的設定檔，並從「Cloud Partner 入口網站」編輯發行者顯示名稱。 


> [!NOTE]  
> 從  **[Cloud Partner 入口網站的 [使用者] 區段](cloud-partner-portal-manage-users.md)**中左導覽窗格上的 [使用者] 區段，將使用者 (參與者和擁有者) 新增至您的發行設定檔。 

**若要檢視及管理您的發行者設定檔**，請從右上方功能表列按一下 [發行者設定檔] 索引標籤。 

![發行者設定檔索引標籤](./media/cloud-partner-portal-how-to-manage-publisher-profile/publisherprofilenew.png)



![發行者設定檔詳細資料](./media/cloud-partner-portal-how-to-manage-publisher-profile/publisherprofiledetails.png)


**連結您的開發中心帳戶**︰您也可以在 Cloud Partner 入口網站上連結現有開發中心帳戶與您的發行者設定檔。 首先以用來註冊開發中心帳戶的相同電子郵件地址登入入口網站。 一旦連結，您的開發中心帳戶狀態、開發中心帳戶擁有者電子郵件和開發中心帳戶名稱就會顯示在您的發行者設定檔頁面上。 

> [!NOTE]
> 發佈付費的 Marketplace SKU 時，必須註冊開發中心帳戶。

如果您沒有**[開發中心帳戶](https://docs.microsoft.com/azure/marketplace-publishing/marketplace-publishing-accounts-creation-registration)**，您可以建立一個帳戶並將已註冊的電子郵件地址新增為 Cloud Partner 入口網站上您供應項目的其中一個擁有者。  擁有者只可以從 [使用者] 區段將已註冊的電子郵件地址新增至發行設定檔。 然後，使用相同電子郵件地址登入入口網站，然後連結您的開發中心帳戶。
