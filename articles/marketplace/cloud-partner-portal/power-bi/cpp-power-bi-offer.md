# <a name="power-bi-application-offer"></a>Power BI 應用程式供應項目

|                  |                                |
|------------------|--------------------------------|
| 本節說明如何將 Power BI 應用程式發佈到 Microsoft [AppSource Marketplace](https://appsource.microsoft.com/)。 Power BI 應用程式封裝 Power BI 內容，並使用資料流程連接在相同資料儲存體中的報表和儀表板。 然後可以透過 AppSource 將應用程式部署到其他租用戶。 | ![Power BI 圖示](./media/powerbi-icon.png) |


本節分為這些主要部分：

-   [必要條件](./cpp-prerequisites.md)：建立或發佈 Power BI 應用程式供應項目在技術上和商務上的必要條件
-   [建立 Power BI 應用程式供應項目](./cpp-create-offer.md)：使用 [Cloud Partner 入口網站](https://cloudpartner.azure.com)建立新 Power BI 應用程式供應項目所需的步驟
-   [發佈 Power BI 應用程式供應項目](./cpp-publish-offer.md)：如何提交要發佈到 AppSource 的新供應項目，以及如何更新現有的供應項目。


## <a name="publishing-steps"></a>發佈步驟

下圖說明大致的 Power BI 應用程式發佈流程。

![Power BI 應用程式發佈步驟](media/publishing-steps.png)

Power BI 應用程式發佈程序

1. 在 Power BI 中建立服務應用程式 (最終會建立套件安裝 URL)。  此 URL 代表供應項目的技術資產。  如需詳細資訊，請參閱[什麼是 Power BI](https://go.microsoft.com/fwlink/?linkid=2028636) \(英文\)。
2. 收集或建立商務資產，包括：
    - 行銷材料
    - 法律和支援文件
3.  建立供應項目：使用 Cloud Partner 入口網站來設定所有供應項目相關的詳細資料，包括供應項目說明、行銷資料、法律、支援資訊和資產的規格。
4.  認證及發佈供應項目：此提交作業會啟動發佈程序。
    - AppSource 上架小組會測試、驗證以及認證您的應用程式。 
    - 通過認證之後，您應在應用程式的測試環境中檢閱它，然後推出應用程式。

執行所有上述步驟之後，您的應用程式就會在 AppSource 上架。
