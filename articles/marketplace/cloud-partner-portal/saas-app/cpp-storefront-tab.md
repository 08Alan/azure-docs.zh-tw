# <a name="saas-application-storefront-details-tab"></a>SaaS 應用程式電子店面詳細資料索引標籤

本文示範如何使用 [電子店面詳細資料] 索引標籤，來說明您的 SaaS 應用程式並提供行銷資產。 此索引標籤包含下列表單：概觀、行銷成品、潛在客戶管理及法律聲明。 


## <a name="overview"></a>概觀

[概觀] 表單包含下一個螢幕擷取畫面所示的必要欄位和選擇性欄位。 欄位名稱上附加的星號 (*) 表示此為必填欄位。

![電子店面概觀表單](./media/saas-storefront-overview.png)

下表描述您可以為供應項目提供的電子店面詳細資料。 必要欄位由星號 (*) 標示。

|     欄位               |      描述       |
|     ------              |      -----------       |
|  **供應項目摘要\***    | 供應項目的價值主張摘要。 它將會出現在您供應項目的搜尋頁面上。 且上限不可超過 100 個字元。  |
| **供應項目描述**  | 將出現在您應用程式詳細資料頁面上的描述。 上限為 1300 個字元。 您可以使用基本 HTML 標記型態的標記來設定內容格式。 例如，&lt;p&gt;、&lt;h1&gt;、&lt;h2&gt; 及 &lt;li&gt;。 若要查看已設定格式之描述的外觀，請使用線上即時 HTML 工具，例如 [http://htmledit.squarefree.com](https://htmledit.squarefree.com)    |
|  **產業**   | 選取您供應項目的目標產業。 若您的應用程式與多個產業相關，您最多可以選取兩個產業。   |
| **建議的分類 （最多 3）\*** | 選取您供應項目的目標類別。 您最多可以選取三個類別。 |
| **應用程式版本**         | 輸入您應用程式的版本號碼。 |
| **搜尋關鍵字 (最多 3 個)** | 輸入最多三個搜尋關鍵字，客戶可以使用這些關鍵字在 Marketplace 店面網站尋找您的應用程式。  |
|   |   |


## <a name="marketing-artifacts"></a>行銷成品

使用**行銷構件**表單，以識別 Azure Marketplace 行銷資產，例如標誌、 影片、 螢幕擷取畫面和文件。

![電子店面行銷成品表單](./media/saas-storefront-artifacts.png)

下表描述行銷成品的欄位。  必要欄位由星號 (*) 標示。

|       欄位       |            描述            |
|       ------      |            -----------            |
|    **標誌**      |  如果這是「透過 Microsoft 銷售」SaaS 應用程式，您應該提供所有標誌影像。 若這是清單，則只需 2 個標誌。 在 Cloud Partner 網站上傳的標誌都應遵循下列指導方針：<br><ul><li>請保持標誌上具有最少的主要和次要色彩數目。 Azure 設計具有簡單的調色盤。 </li><li>避免使用黑色或白色作為標誌的背景色彩。 Azure 入口網站的佈景主題色彩是黑色與白色。 您應該改為使用會讓您的標誌在 Azure 入口網站顯得突出的某些色彩。 建議您使用簡單的主要色彩。 如果您使用透明背景，則要確定標誌與文字不是黑色、白色或藍色。 </li><li>請不要在標誌上使用漸層背景。 </li><li>避免在標誌上放置文字 (甚至是公司或品牌名稱)。 您標誌的外觀與風格應該是「一般」，而且應該避免漸層。</li><li>不應延展標誌影像。</li></ul>    |
|   **影片**       | 可讓您新增的影片，您的供應項目連結。 您可以使用 YouTube 和/或 Vimeo 影片的連結，這會隨著您的供應項目顯示給客戶。 您也需要輸入影片的縮圖影像，它必須是 1280 x 720 像素的 PNG 影像。 您最多可以為每個供應項目新增四個影片。 |
|  **文件**     | 可讓您將行銷文件新增至您的供應項目。 所有文件都必須是 PDF 格式，而且每個供應項目最多可以有三份文件。   |
|  **螢幕擷取畫面**   | 可讓您新增的供應項目的螢幕擷取畫面。 您可以為每個供應項目新增最多五個螢幕擷取畫面。 影像大小上限是 1280 x 720 像素。  |
|  **實用連結**  | 可讓您新增您的供應項目，以協助架構圖表的點或客戶會想要查看其他網站的外部 Url。 |
|  |  |

## <a name="lead-management"></a>潛在客戶管理

若要設定潛在客戶管理，從下拉式清單中選取 [潛在客戶目的地]。 下一個螢幕擷取畫面顯示可用的目的地。

![電子店面詳細資料的潛在客戶管理](./media/saas-storefront-lead-destination.png)

>[!TIP] 
>選取資訊圖示，即可看到此訊息：「選取將儲存您潛在客戶的系統。 請在[這裡](https://docs.microsoft.com/azure/marketplace/cloud-partner-portal-orig/cloud-partner-portal-get-customer-leads)了解如何連線至 CRM 系統。」


## <a name="legal"></a>法律

使用 [法律聲明] 表單來提供每個供應項目所需的法律聲明文件。

![電子店面詳細資料的法律聲明表單](./media/saas-storefront-lead-legal.png)

請提供下列資訊：

- **隱私權原則 URL\***  – 輸入您的應用程式隱私權原則的連結。
- **使用規定\*** – 輸入您的應用程式的使用規定。 客戶必須先接受這些條款，才能試用您的應用程式。
