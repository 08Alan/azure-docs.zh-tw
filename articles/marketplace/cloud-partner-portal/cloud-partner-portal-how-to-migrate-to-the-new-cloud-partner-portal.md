# <a name="how-to-migrate-to-the-new-cloud-partner-portal"></a>如何移轉至新的 Cloud Partner 入口網站

移轉至 Cloud Partner 入口網站是一大進步。 我們已聽到您對於發佈程序的意見反應，並進行一些重大投資，以便輕易使用並改善整個發佈體驗。 
*    支援組織識別碼登入
*    即時欄位驗證以攔截錯誤
*    簡化發佈工作流程

我們努力讓您的供應項目非常容易移轉，所以如果您準備好享受較佳的發佈體驗，請看看您的需要做些什麼。

## <a name="what-do-you-need-to-do"></a>您需要做些什麼？
依照下列步驟進行成功移轉︰

**移轉階段**
|          |          |
|----------|----------|
|1.  判斷您已經準備就緒 | 將供應項目移轉至 Cloud Partner 入口網站之前，請確定您已完全了解發生什麼狀況。<br/><br/>**我是否要移轉所有的供應項目？**<br/>我們今天要移轉所有的虛擬機器供應項目。 這表示將會移轉您擁有的每個草稿、預備或列出的虛擬機器供應項目。 <br/>對於任何其他類型的供應項目 (解決方案範本、試用產品/等等)，它們目前會繼續在舊入口網站上運作。<br/><br/>**上線的供應項目是否會繼續運作？**<br/>是，此移轉不會影響上線的供應項目。 我們也已預先填入您的發行者帳戶資訊，以便您在完成移轉後，可以立即在新的入口網站中進行供應項目的更新。 預備供應項目會變成草稿，而您必須在新的入口網站中將它發佈。<br/><br/>**進行移轉時，可以變更我的供應項目價格嗎？**<br/>否，所有上線的供應項目價格仍必須完全相同，因為我們不支援在此移轉中變更價格。<br/><br/>**如果我的供應項目處於預備/發佈中狀態，怎麼辦？**<br/>如果您的供應項目目前在舊的入口網站上進行發佈或預備發佈，則移轉將會失敗。 當您挑選移轉時間時，請務必確定您的供應項目全都處於已發佈/預備狀態。<br/><br/>**新的角色型存取如何運作？**<br/>在新的入口網站中，您的系統管理員會變成發行設定檔的擁有者，而所有共同管理員都會變成參與者。 如果您想要編輯發行設定檔中的任何項目，您應該移至新的入口網站。 <br/>若要管理使用者，如果您想新增某個人以在舊入口網站上處理供應項目，您要將他們新增為舊入口網站上的共同管理員。 如果您想新增某個人以在新入口網站上處理供應項目，您要將他們新增為新入口網站上的參與者。<br/><br/>**當我移轉時，這就是最後的動作嗎？**<br/>當我們移轉您的供應項目時，我們會讓這些供應項目的所有舊入口網站動作變成唯讀狀態，並阻止透過舊入口網站建立新的供應項目。 您可以從停止的地方繼續編輯和處理所有已移轉的供應項目，但只限在新的入口網站中。|
|2.  提交進行移轉 | 一旦您判斷已準備好規劃移轉，請使用發行者名稱和識別碼在 Cloud Partner 入口網站的 [了解] 頁面上提交進行移轉。![發佈流程圖](./media/cloud-partner-portal-how-to-migrate-to-the-new-cloud-partner-portal/migrate.png)|
|3.  確認日期 | 我們會在每星期二的上午 10 點至下午 1 點 (太平洋標準時間) 移轉供應項目，並要求發佈帳戶的擁有者至少在 24 小時前先確認移轉時間。 如果您需要在不同時間移轉，使其不會干擾您的工作流程，我們將與您一起找出最佳時間。<br/>您應該會收到一封電子郵件通知，告知您已完成移轉，而您現在可以繼續在新的入口網站上作業。|
|4.  歡迎使用，盡情享受！ | 移轉之後，您最好繼續使用新的入口網站。 從現在起，您的供應項目只能在舊入口網站上讀取，而未來所有的建立與編輯內容應該來自新的入口網站！ <br/><br/>**移轉發生問題，我該怎麼做？** 如果移轉失敗，我們將還原變更並重新安排其他時間為您移轉。 如果您在移轉完成後有任何問題，請隨時透過意見反應工具與我們聯繫。|
