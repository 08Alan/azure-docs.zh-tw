# <a name="build-your-landing-page"></a>建置登陸頁面  
公司網站上的登陸頁面是從產生需求的活動到變成市集清單的入口。  

一開始，請先識別您的目標。 請為您的解決方案選擇目標客戶，以及要讓客戶採取的動作。  您的解決方案可能包含下列動作。  
*   試用解決方案  
*   立即取得解決方案  
*   不勝枚舉...  

您可以跨多個行銷策略參考登陸頁面。 您可以使用行銷策略。  
*   活動  
*   網路廣播  
*   社交媒體  
*   白皮書  
*   技術訓練研討會  
*   新聞稿  

您的訊息與行動呼籲越一致，客戶就越容易找到您的解決方案。  

當您已備妥活動計劃時，請建立登陸頁面。 請檢閱下表來充分發揮登陸頁面的效果。  

| 最佳做法 | 避免事項 |  
|:--- |:--- |  
| ***傳達解決方案能解決哪些客戶問題，以及 Azure 如何實現解決方案*** |  不要傳達解決方案如何與 Azure 搭配運作。 |  
| ***建立簡短、易記的 URL*** | 使用長 URL。 長 URL 不容易記住且難以找到。 |  
| ***新增相關視覺化內容，像是客戶推薦影片或解決方案架構*** | 使用太多文字會讓客戶忽略您的內容而停止探索。 |  
| ***建立明確的呼籲將訪客導向您的 Marketplace 目錄頁面*** | 頁面上有太多連結或動作。 |  
| ***將行動呼籲置於標題或明確劃分的區段*** |  在文字段落中內嵌您市集清單的連結。 |  
| ***研究出最佳關鍵字並將頁面最佳化以供搜尋*** | 假設您產品名稱產生的搜尋順位較高。 |  
| ***善用廣告活動的關鍵字*** |  在各個 Web 屬性間如果使用許多不同的關鍵字，將會降低廣告投資的成效。 |  
| ***將相關產品名稱與關鍵字置於 `above the fold`*** | 讓使用者必須捲動才能看到您所展示的產品或解決方案。 |  
| ***依據 Microsoft 商標指導方針使用品牌圖像來驗證您的解決方案，<br />例如「Azure 認證」*** | 使用未經核准的 Microsoft 品牌圖像。 |  

*   如需有關「Azure 認證」徽章的詳細資訊，請參閱「Microsoft Azure 認證指南」頁面：[azure.microsoft.com/support/legal/marketplace/certified-guidelines](https://azure.microsoft.com/support/legal/marketplace/certified-guidelines)。  
*   如需商標的其他資源，請瀏覽「品牌中心」頁面：[microsoft.sharepoint.com/teams/brandcentral](https://microsoft.sharepoint.com/teams/brandcentral)。  
*   如需有關「商標圖案建立工具」的詳細資訊，請瀏覽「商標圖案建立工具」頁面：[logobuilder.partner.microsoft.com](https://logobuilder.partner.microsoft.com)。  
*   如需詳細資訊，請瀏覽「運作方式」頁面：[partner.microsoft.com/membership/how-it-works](https://partner.microsoft.com/membership/how-it-works)。
