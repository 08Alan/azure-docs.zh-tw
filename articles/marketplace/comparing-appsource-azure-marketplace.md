# <a name="comparing-appsource-and-azure-marketplace"></a>比較 AppSource 和 Azure Marketplace  
店面可為獨特的客戶需求提供服務。 店面可讓您依角色設定目標。 您可以根據客戶提供適當的解決方案或服務。  

## <a name="understanding-the-differences-between-the-storefronts"></a>了解店面之間的差異  
選擇店面的第一步，是找出您的供應項目適用於哪些對象。 如果以兩個對象作為目標，則您只需發佈一次，即可在兩個店面銷售。  
 
請考量這些商店的其他優點。   

### <a name="appsource"></a>AppSource  

| 優點 | 詳細資料 | 
|:--- |:--- | 
| ***計費有彈性*** | 提供試用版體驗。 目前未提供支援商務功能的發佈選項。 您可以使用目前的訂購和計費基礎結構，而無須進行任何額外的投資或變更。 | 
| ***促進與其他合作夥伴的往來*** | 將獨立軟體廠商、系統整合商及受控服務提供者與特定實作案例連結。 你們可以合作向新客戶進行銷售。 | 
| ***自動化*** | 將獨立軟體廠商、系統整合商及受控服務提供者與特定實作案例連結。 你們可以合作向新客戶進行銷售。 |  
| ***多種雲端類型*** | 目前不支援 Azure Stack、Azure Government 或區域雲端。 | 
| ***在內容裡呈現給客戶*** | 透過 Microsoft 產品 (例如 Dynamics 365、Power BI 和 Office 365) 的應用程式內體驗，來觸及更多客戶。 |  

### <a name="azure-marketplace"></a>Azure Marketplace  

| 優點 | 詳細資料 | 
|:--- |:--- | 
| ***計費有彈性*** | VM 的「預付型方案」計費選項採用 Microsoft Enterprise 合約 (EA) 或 Web Direct 銷售模型。 定價選項包含供應項目永久免費的「免費層」訂用帳戶。 定價選項也包括「立即試用」訂用帳戶。 「立即試用」訂用帳戶是限時推廣期間內免費，之後 VM 就會轉換成付費訂用帳戶。 BYOL 啟用也是一個可支援您的選項。<br />以這兩種計費選項及下列 VM 供應項目來說，都會直接向客戶收取所有已佈建 Azure 資源的費用。<ul> <li>Azure 應用程式：受控應用程式</li> <li>Azure 應用程式：解決方案範本</li> </ul> | 
| ***促進與其他合作夥伴的往來*** | 您目前無法將服務提供者或交付合作夥伴與供應項目連結。 這項功能會在 2018 年推出。 | 
| ***自動化*** | 您目前無法將服務提供者或交付合作夥伴與供應項目連結。 透過附加元件佈建利用自動化 SaaS。 使用「Azure 應用程式：解決方案」範本將 SaaS 型資料收集和部署案例自動化。 |  
| ***多種雲端類型*** | 使用下列類型同時在公用雲端和內部部署解決方案上進行發佈。<ul> <li>Azure Stack</li> <li>Azure Government</li> <li>區域雲端，包括中國和德國</li> </ul> | 
| ***在內容裡呈現給客戶*** | 使用虛擬機器和「Azure 應用程式：解決方案」範本，以在 Azure 入口網站內的內容相關式搜尋體驗中提供您的解決方案。 |
