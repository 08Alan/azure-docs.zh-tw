# <a name="azure-marketplace-participation-policies"></a>Azure Marketplace 參與原則

這些 Microsoft Azure Marketplace 參與原則適用於 Microsoft Azure Marketplace 中所有的發行者和供應項目。 這些原則是 Microsoft Marketplace 發行者合約訂定之條款和條件的補充。 若要參與 Azure Marketplace，發行者皆須一律遵守此文件中說明和/或參考的原則。 如果發行者在任何指定時間無法達成所有條款和條件，Microsoft 可以從 Azure Marketplace 移除發行者的供應項目。 我們可能會不定期更新此文件。

    * *可部署至 Microsoft Azure*：發行者必須在其供應項目的說明資訊中說明如何在 Microsoft Azure 上部署該軟體或服務。
    * *整合或擴充 Microsoft Azure 服務*：在其供應項目清單資訊中，發行者必須說明軟體或服務可整合或擴充哪個 Azure 服務。 軟體或服務如何整合或擴充 Azure 服務。

## <a name="publishing-offers"></a>發佈供應項目

1. 發行者必須在執行 Microsoft Marketplace 發行者合約的 60 天內，在 Azure Marketplace 中至少發行一個供應項目。

1. 發行者必須遵守 Marketplace Publication Guideline 中所定義，且可能在發行入口網站中進一步闡述的 Azure Marketplace 上架技術需求。

## <a name="offer-listings"></a>供應項目清單

1. 發行者必須在其供應清單頁面上提供詳細的供應項目資訊，資訊必須是正確和最新的。 這些資訊必須包括 (如適用)：
    * *供應項目描述*
        * 最低供應項目描述
            * SKU 資訊
            * 價值主張
        * 建議的供應項目描述
            * 詳細的 SKU 資訊
            * 詳細的價值主張
            * 功能：3-5 項關於供應項目的真實敘述
            * 優點：3-5 項供應項目功能產生的結果
    * *定價模式*：供應項目必須與本文件[定價模式](./marketplace-faq-publisher-guide.md)一節所說明的 Azure Marketplace 支援的**定價模式**相容。
    * *連結至客戶支援詳細資料*：發行者必須為其在 Azure Marketplace 中的供應項目提供商業上合理的客戶支援。  他們可將其計入與供應項目相關聯的使用者費用中，或將其列為只能個別購買的支援供應項目。
    * *供應項目資源*：資源包括 (但不限於) 示範影片、螢幕擷取畫面、白皮書、個案研究、證明書，以及如何在 Microsoft Azure 上使用發行者供應項目的詳細技術文件。
    * *客戶退費原則*
    * *使用規定*
    * *隱私權原則*

1. 發行者不可在其供應項目清單頁面中將 Azure 客戶重新導向至或向上銷售不在 Azure Marketplace 中的軟體或服務。 此限制不適用於發行者在 Azure Marketplace 之外銷售的支援服務。

1. 發行者不可在 Azure Marketplace 中促銷其在其他雲端平台上的供應項目。

1. 基於品質保證，Microsoft 有權編輯和修改供應項目清單頁面詳細資料。 如果 Microsoft 要對任何清單頁面詳細資料做任何變更，Microsoft 在 Azure Marketplace 的該供應項目清單頁面中發佈資訊之前會通知發行者。

## <a name="offering-classification"></a>供應項目分類

|     |    |
| :------------------- | :-------------------|
| **虛擬機器映像** | 預先配置的虛擬機器 (VM) 映像，包含完整安裝的作業系統以及一或多個應用程式。 虛擬機器映像供應項目包含單一 VM 映像，或是由 Resource Manager 範本繫結在一起的多個 VM 映像。 <br> <br> 虛擬機器映像 (「映像」) 提供在 Azure 虛擬機器服務中建立及部署虛擬機器所需的資訊。 「映像」由作業系統、虛擬硬碟，和零或多個資料磁碟虛擬硬碟所組成。 客戶可以從單一「映像」部署任意數目的虛擬機器。 |
| **虛擬機器擴充功能** | 可透過多種方式新增至新 VM 的 VM 代理程式，包括經由 REST API、Azure 入口網站或 Azure PowerShell Cmdlet。 VM 擴充功能可以手動安裝在現有 VM 上，也可以針對 Windows Server 或 Linux 型 VM 設定。 <br> <br> 虛擬機器擴充功能 (「**VM 擴充功能**」) 是一種機制，用以在 Azure 虛擬機器內安裝軟體應用程式或軟體應用程式套件。 VM 擴充功能可包含軟體應用程式。  在虛擬機器內安裝或執行之後，它可能會下載及安裝一或多個來自外部位置的軟體應用程式。 也就是說，您的 VM 擴充能所安裝的任何軟體或其他資料，即使是從外部位置擷取，也會被視為此合約特定用途的「供應內容」。 您有責任且必須向客戶提供與您的 VM 擴充功能供應項目相關聯的 VM 擴充功能處理常式支援。 |
| **服務** | 完全受控的服務，供資訊工作者、商業分析師、開發人員或 IT 專家用於自訂應用程式開發部署或系統管理。 Marketplace 支援三種服務： <br> <br> **應用程式服務**提供的功能，可讓客戶在 Azure 上快速開發雲端規模的應用程式。 <br> <br> 客戶必須有 Azure 訂用帳戶才能購買應用程式服務。 如 Microsoft Marketplace 發行者合約所說明，發行者須負責測量客戶的應用程式服務使用量，以及向 Microsoft 回報使用量資訊。|
| **Web 應用程式** | 應用程式套件，可在 Azure 網站服務中安裝和部署開放原始碼或專屬網站內容或管理平台。 Web 應用程式必須符合 Microsoft [Web 應用程式庫準則](https://www.iis.net/learn/develop/windows-web-application-gallery/windows-web-application-gallery-principles)。 <br> <br> 在此合約中，「**Web 應用程式**」是指客戶用來在 Azure 網站服務中安裝及部署開放原始碼或專屬網站應用程式的應用程式套件。 |
| **目錄清單** | 非直接透過 Marketplace 提供給 Azure 客戶的供應項目，但 Marketplace 會顯示連結、圖示和軟體/服務產品清單。  客戶會被導向至發行者的網站，或獲得有關於如何在 Azure 上取得和使用供應項目的指示。 「**僅限目錄清單**」是指不直接透過 Marketplace 提供給客戶的供應項目。 此時，Marketplace 中會顯示連結、圖示及/或描述，將客戶導向至您的網站。 此外也會提供有關於客戶如何在 Azure 中取得及使用供應項目的指示。 也就是說，「僅限目錄清單」所參考的任何軟體或資料，均視為此合約的「供應內容」。 |
| **Azure Resource Manager (Resource Manager) 範本** | Resource Manager 範本可參考多個不同的供應項目，包括其他發行者所發佈的供應項目。  此範本可讓 Azure 客戶以單一、協調的方式部署一或多個供應項目。 <br> <br> 「**Azure Resource Manager 範本**」是參考一或多個「供應項目」的資料結構，並且包含「供應項目」的相關中繼資料。  資料結構與清單資訊相關聯。 Marketplace 服務會使用 Resource Manager 範本來顯示特定類別的供應項目，供客戶部署。 發行者可以在 Marketplace 中發佈參考多個不同供應項目的 ARM 範本，包括其他發行者所發佈的供應項目。 |
| | |

如果您想要發佈的應用程式或服務不屬於任何已討論過的類別，請在 [Azure Marketplace 論壇](https://feedback.azure.com/forums/216369-azure-marketplace)中輸入要求。

## <a name="pricing-models"></a>定價模式

下表說明 Azure Marketplace 目前支援的定價模式。 供應項目可能包含使用不同定價模式的不同 SKU。

| **定價模式** | **說明** | **適用於** |
| :---------------- | :---------------| :-----------------|
| **免費** | 免費 SKU。 客戶使用供應項目時不會被收取 Azure Marketplace 費用。 免費 SKU 的價格不可增加至非零金額。 | <ul><li>虛擬機器映像</li><li>VM 擴充功能</li><li>服務</li><li>Resource Manager 範本</li></ul> |
| **免費試用版** <br> **(立即試用)** | 限定期間內的免費促銷 SKU。 客戶在試用期間使用供應項目不會被收取 Azure Marketplace 費用。 試用到期後，將自動根據使用供應項目的標準費率向客戶收費。 Marketplace 目前無法防止客戶建立多個免費試用版供應項目的訂用帳戶。 若發行者想要限制客戶為免費試用版供應項目建立的訂用帳戶數目，須負責在其「使用規定」中加入適當的限制。 | <ul><li>虛擬機器映像</li><li>服務</li></ul> |
**BYOL** | 自備授權 (BYOL) SKU。 客戶若在 Azure Marketplace 以外取得存取或使用供應項目的權利，將不會被收取 Azure Marketplace 費用。 | 虛擬機器映像 |
| **每月訂用帳戶** | 客戶訂閱供應項目會被收取固定月費。 除了下列說明外，每月訂用帳戶自客戶購買日期起算。 月費不依客戶月中取消或未使用的服務而按比例攤提。 若客戶的授權條款要求照日曆月份計費，或客戶在月中升級或降級其訂用帳戶，則會按比例計算月費。 只有在發行者將供應項目設定為支援升級和降級，才能執行這項作業。 | 服務 |
| **依使用量** | 根據供應項目的使用程度向客戶收費。 針對虛擬機器映像，如發行者所訂，客戶若使用從映像部署的虛擬機器，將依小時向客戶收取 Azure Marketplace 費用。 每小時的費用依虛擬機器的大小而採均一價或變動價。 不到一小時以分鐘計。 <br><br> 針對應用程式服務，為方便計費，發行者須負責定義測量單位。 (例如交易次數、傳送的電子郵件數目等等。)發行者可為相同應用程式服務方案定義多個計量。 發行者須負責以每個供應項目所定義的計量追蹤個別客戶的使用量。 他們也需使用 Microsoft 所提供的報告機制，向 Microsoft 報告此追蹤資訊 (以小時為單位)。 Microsoft 會根據發行者就適用計費期間報告的使用量資訊向客戶收費。 | <ul><li>服務</li><li>虛擬機器映像</li></ul> |
| | | |

現在，發行者可以針對已發佈的虛擬機器供應項目降低其使用者費用。 在 Marketplace 中，目前並不支援對現有的供應項目進行任何其他類型的價格變動。 發行者若想要變更與供應項目相關聯的使用者費用，必須先從 Marketplace 中移除該供應項目。  移除作業應依據 Microsoft Marketplace 發行者合約和本文件的需求進行。  其後，發行者可發佈包含新使用者費用的新供應項目。

在 Marketplace 上發佈服務供應項目後，發行者必須保留其本身的資料記錄 (至少保留前兩 (2) 個日曆年份)。 此資料記錄包含他們為客戶佈建的供應項目。 如果發行者與 Microsoft 的資料記錄間有任何衝突，將以 Microsoft 的資料記錄為準。

## <a name="offering-suspension-and-removal"></a>供應項目的擱置和移除

1. Microsoft 有權基於任何原因從 Azure Marketplace 擱置或移除供應項目。 Microsoft 會移除供應項目的原因包括 (但不限於)：
    * 供應項目己超過 6 個月無客戶佈建；
    * 供應項目的付費 SKU 取消率偏高；
    * 供應項目一直得到客戶的負評；
    * 供應項目一直收到大量支援票證；或
    * 發行者未遵守 Microsoft Marketplace 發行者合約、Marketplace Publication Guidelines 或本文件的條款與條件。
