# Azure 金鑰保存庫的可用性與備援

Azure 金鑰保存庫具備多層備援功能，可確保您的金鑰和密碼會保持可供應用程式使用，甚至在服務的個別元件失敗時也是如此。

金鑰保存庫的內容會在區域內複寫，以及複寫到至少有 150 英哩距離的次要地區 (但在相同地理位置內)。這可確保您的金鑰和密碼具有高持久性。

如果金鑰保存庫服務內的個別元件失敗，那麼，區域內替代的元件會接替來處理您的要求，以確保不會導致功能的效能降低。您不需要採取任何動作來觸發此動作 — 它將會自動發生，而且對您而言是透明的。

在整個 Azure 區域無法使用的罕見情況下，您在該區域中的金鑰保存庫進行的要求會自動路由傳送 (「容錯移轉」) 到次要地區。當主要區域再次可用時，要求會路由傳送回 (「回復」) 到主要區域。同樣地，您不需要採取任何動作，因為這會自動發生。

有一些您應該要注意的警告事項：

* 若發生區域容錯移轉，可能需要幾分鐘，服務才能進行容錯移轉。容錯移轉完成之前，在這段期間所做的要求可能會失敗。
* 容錯移轉完成之後，金鑰保存庫會處於__唯讀__模式。在此模式中支援的要求是：
 * 列出金鑰保存庫
 * 取得金鑰保存庫的屬性
 * 列出密碼 
 * 取得密碼
 * 列出金鑰
 * 取得金鑰 (的屬性)
 * 加密
 * 解密
 * 包裝
 * 解除包裝
 * 驗證
 * 簽署
 * backup
* 一旦容錯移轉回復，所有要求的類型 (即讀取___和___寫入要求) 均可供使用。
