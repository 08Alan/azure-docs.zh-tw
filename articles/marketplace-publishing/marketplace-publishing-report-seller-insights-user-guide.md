# <a name="how-to-use-the-microsoft-azure-marketplace-seller-insights-portal"></a>如何使用 Microsoft Azure Marketplace Seller Insights Portal
##<a name="onboarding"></a>登入

>[!NOTE]
>僅限系統管理員

###<a name="overview-of-roles"></a>角色概觀

![帳戶類型][5]

Seller Insights Portal 系統管理員可以新增使用者、編輯使用者，以及刪除使用者。

Seller Insights Portal 使用者可以檢視可用於您組織的所有資料。

需要有系統管理員和使用者，才能使用有效 Microsoft 帳戶 (LiveID http://signup.live.com) 或公司帳戶進行驗證。 請確定輸入的識別碼是與受監視的信箱相關聯。

###<a name="invitation-to-activate-your-account"></a>啟用帳戶的邀請

在理想情況下，登入 Seller Insights Portal 之前，系統管理員應該識別他們想要設定為使用者之個人的帳戶。 帳戶可以是 Microsoft 帳戶或公司行號。

為了啟用服務，系統管理員應該瀏覽至邀請電子郵件中所列的 URL (https://reports.azure.com)

>[!NOTE]
>附註︰如果系統管理員電子郵件列為 Microsoft 帳戶，而且您尚未建立接收邀請之電子郵件地址相關的 Microsoft 帳戶，則需要先在登入之前這麼做，方法是在建立 Microsoft 帳戶時利用此電子郵件地址瀏覽至 https://signup.live.command。

###<a name="sign-in-to-the-portal"></a>登入 Portal

1. 按一下傳送給您電子郵件中的 URL，以登入 Seller Insights Portal。
2. 輸入系統管理員電子郵件地址和密碼。
3. 按一下 [登入] 按鈕。
4. 開始管理您的帳戶 – 新增使用者、設定權限，以及編輯使用者。
5. 請務必將 Portal 的 URL 提供給使用者。

![登入頁面][6]

##<a name="signing-in"></a>登入

系統管理員應該透過電子郵件通知您已建立您的帳戶，並與您分享 Seller Insights Portal 的 URL (https://reports.azure.com)。

>[!NOTE]
>系統管理員是實際將解決方案發佈到 Azure Marketplace 之帳戶的擁有者。  此帳戶的擁有者可以將存取權提供給其他人。  

您可以使用個人 Microsoft 帳戶或使用公司帳戶登入 Seller Insights Portal，取決於設定您的設定檔時系統管理員使用的帳戶。  您的系統管理員應該通知您要使用哪一個帳戶。

![帳戶類型][1]



***歡迎使用 Seller Insights Portal！***

1. 一旦登入，即會登陸在 [摘要] 頁面上。
2. 請花幾分鐘的時間來閱讀隱私權原則。
3. 若有與 Seller Insights Portal 相關的問題，您隨時可以透過電子郵件「與我們連絡」。
4. 您可以使用頁面頂端的索引標籤來瀏覽 Seller Insights Portal。  尋找本指南頁面頂端的索引標籤，可協助您熟悉 Seller Insights Portal。
5. 隨時按一下笑臉符號可傳送與 Seller Insights Portal 相關的意見反應。
6. 隨時在右上角您的名稱上按一下，可登出 Seller Insights Portal。

![預設登陸頁面][2]


##<a name="summary-view"></a>摘要檢視

1. 選取 [摘要] 索引標籤。
2. 在這裡輸入您想要之期間的開始日期，然後按一下 [套用]。 將顯示 3 個月的資料。
3. 會在這裡顯示所選取 3 個月期間的新「訂單」趨勢資料。
4. 將會依 [通道] 和 [優惠類型] 在這裡顯示所選取期間的新訂單。
5. 從這個面板中，選取 [前 5 個市場]、[前 5 個執行優惠] 或 [前 5 個客戶] 的檢視。
6. 在這裡會顯示付費優惠之所選取期間的趨勢。 如果您只有 BYOL 或第 1 方優惠，則不會顯示此面板。

![摘要檢視][7]

##<a name="orders-and-usage"></a>訂單和使用量
###<a name="overview"></a>概觀

A. 在 [面板 A] 中，您可以選取 [訂單和使用量] 索引標籤。  在這裡，您可以從 [報告類型] 索引標籤選取 [訂單] 或 [使用量]，並使用滑桿選取想要的資料。

B. 在 [面板 B] 中，您可以選取每月資料 (根據 [面板 A] 之 [報告類型] 中的選取項目，可能是 [訂單] 或 [使用量])。呈現的資料會反映您所輸入的日期，以及使用 [面板 A] 中的滑桿所套用的篩選條件。

C. 在 [面板 C] 中，您可以選取 3 個月的趨勢資料 (根據 [面板 A] 之 [報告類型] 中的選取項目，可能是 [訂單] 或 [使用量])。呈現的資料會反映您所輸入的日期，以及使用 [面板 A] 中的滑桿所套用的篩選條件。

D. *面板 D*︰此面板提供與每個個別訂用帳戶相關的資料。  您可以使用 Excel 或 csv 格式，從此面板下載資料。

##<a name="user-management"></a>使用者管理

>[!NOTE]
>僅限系統管理員

###<a name="add-user"></a>新增使用者

1. 選取 [使用者管理] 索引標籤。
2. 若要新增使用者，請瀏覽至 [使用者管理]，然後 [新增使用者]。
3. 新增使用者，方法是輸入與其 Microsoft 帳戶或公司帳戶相關聯的電子郵件地址，然後按一下 [繼續]。
4. 身為系統管理員，您可以指派使用者的唯讀報告存取權。
5. 輸入使用者的詳細資料 ([名字]、[姓氏] 和 [電子郵件])，然後按一下 [繼續]。
6. 按一下您想要配置給使用者的權限，然後按一下 [繼續]。
7. 確認您所輸入的資訊，然後按一下 [儲存] 繼續。  您已經成功新增使用者。

![新增使用者面板][29]


###<a name="edit-user"></a>編輯使用者

1. 選取 [使用者管理] 索引標籤，然後選取 [編輯使用者]。
2. 尋找您想要修改其權限的使用者。  請注意，您也可以從這裡 [新增使用者]。
3. 按一下您要為使用者移除的權限。  請注意，您也可以從這裡 [刪除使用者]。
4. 按一下 [儲存] 繼續。  您已成功編輯使用者的權限。

![編輯使用者面板][30]

##<a name="support-requests"></a>支援要求


###<a name="access-the-support-request-format"></a>存取支援要求格式

1. 選取 [說明] 索引標籤。
2. 選取 [支援和文件] 和 [啟動要求]。
3. 您也可以從應用程式中每個畫面底部的 [與我們連絡] 連結，來提交支援要求。
4. 將會開啟新的索引標籤，以讓您提交支援要求。

![存取支援表單][31]

###<a name="review-helpful-information"></a>檢閱有用資訊

1. 若要提交與報告有關的支援要求，請從 [問題類型] 中選取 [報告和見解]。
2. 選取類別。  選項是 [與報告相關的問題] 和 [其他]。
3. 會顯示一些有用的連結，以提供查詢的答案。  請先檢查這些項目，再提交支援要求。
4. 如果您仍然需要提交支援要求，請選取 [啟動要求]。

![編輯使用者面板][32]


###<a name="fill-out-and-submit-the-request"></a>填寫並提交要求

1. 輸入您的連絡資訊、時區、區域和語言喜好設定，然後選取 [繼續]。
2. 提供您事件的標題，並盡可能提供最多資訊。  您也可以上傳支援檔案與螢幕擷取畫面等等，然後選取 [提交]。
3. 您會收到確認訊息，指出已成功提交支援要求。
4. 您隨時可以檢查支援要求的狀態。

![存取支援表單][33]


##<a name="resetting-a-password"></a>重設密碼

***帳戶和密碼驗證***

1. ***驗證您使用的帳戶已註冊，否則密碼無法運作。*** 如果您輸入已在 Azure Active Directory 中註冊的帳戶，它將會自動提示您選擇如何使用帳戶登入 - 工作或學校 (組織帳戶) 或個人 LiveID 帳戶。
2. 瀏覽至 https://reports.azure.com 並從您可能已登入的帳戶登出，或在隱私視窗中開啟該網址，以確保未使用其他帳戶登入。
3. 輸入發行者電子郵件地址，不需要輸入密碼。 如果該帳戶已在 Azure Active Directory 中登錄，則會重新導向至***步驟 4*** (系統管理員帳戶已登錄，因為任何組織帳戶是由系統管理員帳戶設定)。
4. 選擇 [公司帳戶或學校帳戶]。 工作或學校帳戶 (「組織帳戶」) 是由已向 Azure 登錄的公司設定；個人帳戶 ('LiveID') 是已向 Microsoft 登錄的帳戶。
 * 請注意，使用者可能先前已使用其公司電子郵件地址設定 LiveID 帳戶，後來公司上架至 Azure，並且將相同的電子郵件地址登錄為組織帳戶。 這表示此使用者可以有一個 LiveID 並具備其用來建立 LiveID 的密碼，以及一個組織帳戶並具備與公司所使用的不同密碼。
5. 針對 Seller Insights Portal，選取 [工作或學校帳戶] 時，請確定您是使用組織帳戶/密碼組合，而非 LiveID/密碼組合。
6. 針對組織帳戶，這樣會將重新導向至發行者登入頁面。
7. 從這裡開始，請驗證組織帳戶的密碼是否運作。 如果無法運作，請繼續進行 [密碼重設]。

***密碼重設***

請遵循下列連結，嘗試使用自助密碼重設精靈來重設您的密碼。

>[!NOTE]
>注意︰針對系統管理員帳戶設定的組織帳戶，如果系統管理員開啟可讓您重設密碼的功能，您將能夠重設自己的密碼。 否則，密碼重設表單可讓您連絡系統管理員。
