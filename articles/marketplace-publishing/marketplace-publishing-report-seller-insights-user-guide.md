# <a name="how-to-use-the-microsoft-azure-marketplace-seller-insights-portal"></a>如何使用 Microsoft Azure Marketplace Seller Insights Portal

##<a name="signing-in"></a>登入

系統管理員應該透過電子郵件通知您已建立您的帳戶，並與您分享 Seller Insights Portal 的 URL (https://reports.azure.com)。

您可以使用個人 Microsoft 帳戶或使用公司帳戶登入 Seller Insights Portal，取決於設定您的設定檔時系統管理員使用的帳戶。  您的系統管理員應該通知您要使用哪一個帳戶。

   ![帳戶類型][1]



***歡迎使用 Seller Insights Portal！***

1. 一旦登入，即會登陸在 [摘要] 頁面上。
2. 請花幾分鐘的時間來閱讀隱私權原則。
3. 若有與 Seller Insights Portal 相關的問題，您隨時可以透過電子郵件「與我們連絡」。
4. 您可以使用頁面頂端的索引標籤來瀏覽 Seller Insights Portal。  尋找本指南頁面頂端的索引標籤，可協助您熟悉 Seller Insights Portal。
5. 隨時按一下笑臉符號可傳送與 Seller Insights Portal 相關的意見反應。
6. 隨時在右上角您的名稱上按一下，可登出 Seller Insights Portal。

   ![預設登入頁面][2]



##<a name="resetting-a-password"></a>重設密碼

***帳戶和密碼驗證***

1. ***驗證您使用的帳戶已註冊，否則密碼無法運作。*** 如果您輸入已在 Azure Active Directory 中註冊的帳戶，它將會自動提示您選擇如何使用帳戶登入 - 工作或學校 (組織帳戶) 或個人 LiveID 帳戶。
2. 瀏覽至 https://reports.azure.com 並從您可能已登入的帳戶登出，或在隱私視窗中開啟該網址，以確保未使用其他帳戶登入。
3. 輸入發行者電子郵件地址，不需要輸入密碼。 如果該帳戶已在 Azure Active Directory 中登錄，則會重新導向至***步驟 4*** (系統管理員帳戶已登錄，因為任何組織帳戶是由系統管理員帳戶設定)。
4. 選擇 [公司帳戶或學校帳戶]。 工作或學校帳戶 (「組織帳戶」) 是由已向 Azure 登錄的公司設定；個人帳戶 ('LiveID') 是已向 Microsoft 登錄的帳戶。
 * 請注意，使用者可能先前已使用其公司電子郵件地址設定 LiveID 帳戶，後來公司上架至 Azure，並且將相同的電子郵件地址登錄為組織帳戶。 這表示此使用者可以有一個 LiveID 並具備其用來建立 LiveID 的密碼，以及一個組織帳戶並具備與公司所使用的不同密碼。
5. 針對 Seller Insights Portal，選取 [工作或學校帳戶] 時，請確定您是使用組織帳戶/密碼組合，而非 LiveID/密碼組合。
6. 針對組織帳戶，這樣會將重新導向至發行者登入頁面。
7. 從這裡開始，請驗證組織帳戶的密碼是否運作。 如果無法運作，請繼續進行***密碼重設***。

***密碼重設***

請遵循下列連結，嘗試使用自助密碼重設精靈來重設您的密碼。
 * 注意︰針對系統管理員帳戶設定的組織帳戶，如果系統管理員開啟可讓您重設密碼的功能，您將能夠重設自己的密碼。 否則，密碼重設表單可讓您連絡系統管理員。

****Microsoft 帳戶 (Live ID)：https://account.live.com/ResetPassword.aspx****    ![Microsoft 帳戶密碼重設][3]
