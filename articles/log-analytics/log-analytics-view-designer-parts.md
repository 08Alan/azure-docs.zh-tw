# <a name="log-analytics-view-designer-visualization-part-reference"></a>Log Analytics 檢視設計工具視覺效果部分參考
Log Analytics 中的檢視設計工具可讓您在 OMS 主控台中建立自訂檢視，其中包含 OMS 存放庫中不同資料的視覺效果。 本文針對可用於自訂檢視的每個視覺效果部分提供其設定參考。

其他與檢視設計工具相關的文章︰

- [檢視設計工具](log-analytics-view-designer.md) -檢視設計工具的概觀以及建立和編輯自訂檢視的程序。
- [圖格參考](log-analytics-view-designer-tiles.md) - 可用於自訂檢視之每個圖格的設定參考。 

下表說明檢視設計工具中可用的各種圖格類型。  下列各節詳細說明每種圖格類型及其屬性。

| 檢視類型 | 說明 |
|:--|:--|
| [查詢清單](#list-of-queries-part) | 顯示記錄檔搜尋查詢清單。  使用者可以按一下每個查詢，以顯示其結果。  根據預設，檢視會包含單一查詢，您可以按一下 [+ 查詢] 來新增額外的查詢。 |
| [數字與清單](#number-amp-list-part) | 標頭中有一個數字可顯示記錄檔搜尋查詢中的記錄計數。  清單會顯示查詢的前 10 筆結果，其中的水平列表示數值資料行的相對值。 |
| [兩個數字與清單](#two-numbers-amp-list-part) | 標頭中有兩個數字可顯示不同記錄檔搜尋查詢中的記錄計數。  清單會顯示查詢的前 10 筆結果，其中的水平列表示數值資料行的相對值。 |
| [環圈與清單](#donut-amp-list-part) | 標頭會顯示從記錄檔查詢中的值資料行摘錄而來的單一數字。  環圈會以圖形方式顯示前三筆記錄的結果。 |
| [兩個時間軸與清單](#two-timelines-amp-list-part) | 標頭會將一段時間內兩個記錄檔查詢的結果顯示為直條圖，其圖說文字會顯示從記錄檔查詢中的值資料行摘錄而來的單一數字。  清單會顯示查詢的前 10 筆結果，其中的水平列表示數值資料行的相對值。 |   
| [資訊](#information-part) | 標頭會顯示靜態文字和選擇性連結。  清單會顯示一或多個包含靜態文字和標題的項目。 |
| [折線圖、圖說文字與清單](#line-chart-callout-amp-list-part) | 標頭會顯示一個折線圖 (其中包含一段時間內記錄檔查詢中的多個系列)，以及包含摘錄值的圖說文字。  清單會顯示查詢的前 10 筆結果，其中的水平列表示數值資料行的相對值。 |
| [折線圖與清單](#line-chart-amp-list-part) | 標頭會顯示一個折線圖，其中包含一段時間內記錄檔查詢中的多個系列。  清單會顯示查詢的前 10 筆結果，其中的水平列表示數值資料行的相對值。 |
| [折線圖與走勢圖](#line-chart-amp-sparklines-part) | 標頭會顯示一個折線圖，其中包含一段時間內記錄檔查詢中的多個系列。  清單會顯示查詢的前 10 筆結果，其中的走勢圖會將所選時間範圍的數值資料行值視覺化。 |


## <a name="list-of-queries-part"></a>查詢清單部分

顯示記錄檔搜尋查詢清單。  使用者可以按一下每個查詢，以顯示其結果。  

![查詢清單檢視](media/log-analytics-view-designer/view-list-queries.png)

| 設定 | 說明 |
|:--|:--|
| **一般** |
| 標題 | 顯示於檢視頂端的文字。 |
| 新增群組 | 選取即可在檢視中建立新群組 (開始於目前檢視)。 |
| 預先選取的篩選器 | 當使用者選取查詢時，要納入左側篩選窗格中的屬性清單 (以逗號分隔)。 |
| 轉譯模式 | 選取查詢時顯示的初始檢視。  使用者可以在開啟查詢後，選取任何可用的檢視。 |
| **查詢** |
| 搜尋查詢 | 要執行的查詢。 |
| 易記名稱 | 要對使用者顯示的查詢描述性名稱。 |


## <a name="number-&-list-part"></a>數字與清單部分

標頭中有一個數字可顯示記錄檔搜尋查詢中的記錄計數。  清單會顯示查詢的前 10 筆結果，其中的水平列表示數值資料行的相對值。

![查詢清單檢視](media/log-analytics-view-designer/view-number-list.png)

| 設定 | 說明 |
|:--|:--|
| **一般** |
| 群組標題 | 顯示於檢視頂端的文字。 |
| 新增群組 | 選取即可在檢視中建立新群組 (開始於目前檢視)。 |
| 圖示 | 顯示於標頭中結果旁邊的影像檔。
| 使用圖示 | 選取即可顯示圖示。 |
| **標題** |
| 圖例 | 顯示於標頭頂端的文字。 |
| 查詢 | 針對標頭執行的查詢。  將會顯示查詢所傳回的記錄計數。 |
| **清單** |
| 查詢 | 針對清單執行的查詢。  將顯示結果中前 10 筆記錄的前兩個屬性。  第一個屬性應該是文字值，而第二個屬性是數值。  根據數值資料行的相對值，自動建立長條圖。<br><br>在查詢中使用 Sort 命令，將清單中的記錄排序。  使用者可以按一下 [檢視全部]，以執行查詢並傳回所有記錄。 |
| 色彩 | 長條的色彩。 |
| 名稱和值分隔符號 | 您要將 text 屬性剖析成多個值時的單一字元分隔符號。  如果您指定分隔符號，您可以在 [名稱] 方塊中提供每個欄位的名稱 (以相同的分隔符號分隔)。<br><br>例如，考慮名為 location 的屬性，其包含 Redmond-Building 41 和 Bellevue-Building12 等值。  您可以針對 [名稱與值分隔符號] 指定 –，針對 [名稱] 指定 City-Building。  這會將每個值剖析成兩個名為 City 和 Building 的屬性。 |
| 瀏覽查詢 | 當使用者選取清單中項目時所要執行的查詢。  使用 {selected item} 來包含使用者所選項目的語法。 <br><br>例如，如果查詢具有名為 Computer 的資料行且瀏覽查詢是 {selected item}，則會在使用者選取電腦時執行 Computer=”MyComputer” 之類的查詢。  如果瀏覽查詢是 Type=Event {selected item}，則會執行 Type=Event Computer=”MyComputer”。 |
| **清單** | **> 資料行標題** |
| 名稱 | 顯示於清單第一欄頂端的文字。 |
| 值 | 顯示於清單第二欄頂端的文字。 |

## <a name="line-chart,-callout,-&-list-part"></a>折線圖、圖說文字與清單部分

標頭會顯示一個折線圖 (其中包含一段時間內記錄檔查詢中的多個系列)，以及包含摘錄值的圖說文字。  清單會顯示查詢的前 10 筆結果，其中的水平列表示數值資料行的相對值。

![折線圖、圖說文字與清單檢視](media/log-analytics-view-designer/view-line-chart-callout-list.png)

| 設定 | 說明 |
|:--|:--|
| **一般** |
| 群組標題 | 顯示於圖格頂端的文字。 |
| 新增群組 | 選取即可在檢視中建立新群組 (開始於目前檢視)。 |
| 圖示 | 顯示於標頭中結果旁邊的影像檔。 |
| 使用圖示 | 選取即可顯示圖示。 |
| **標頭** |
| 標題 | 顯示於標頭頂端的文字。 |
| 副標題 | 顯示於標頭頂端之標題下的文字。 |
| **折線圖** |
| 查詢 | 針對折線圖執行的查詢。  第一個屬性應該是文字值，而第二個屬性是數值。  這通常是使用 **measure** 關鍵字的查詢來彙總結果。  如果查詢使用 **interval** 關鍵字，則圖表的 X 軸會使用此時間間隔。  如果查詢不包含 **interval** 關鍵字，則 X 軸是每小時的間隔。 |
| **折線圖** | **> 圖說文字** |
| 圖說文字標題 | 顯示在圖說文字值之上的文字。 |
| 系列名稱 | 將用於圖說文字值之系列的屬性值。  如未提供任何系列，則會使用查詢中的所有記錄。 |
| 作業 | 對值屬性執行作業以彙總出圖說文字的單一值。<br><br>- 平均︰所有記錄值的平均值。<br>- 計數︰查詢所傳回的所有記錄計數。<br>- 最後一個範例︰圖表內含最後一個間隔中的值。<br>- 最大值︰圖表內含間隔中的最大值。<br>- 最小值︰圖表內含間隔中的最小值。<br>- 加總︰所有記錄值的加總。 |
| **折線圖** | **> Y 軸** |
| 使用對數刻度 | 若選取，會在 Y 軸使用對數刻度。 |
| Units | 指定查詢所傳回之值的單位。  這項資訊用來顯示圖上的標籤，指出值的類型，並選擇性用來將值轉換。  單位類型指定單位的分類，並定義可用的 [目前單位類型] 值。  如果您選取 [轉換成] 中的值，則會將數值從 [目前單位] 類型轉換為 [轉換成] 的類型。 |
| 自訂標籤 | Y 軸的顯示文字，就在單位類型的標籤旁。  如未指定任何標籤，只會顯示單位類型。 |
| **清單** |
| 查詢 | 針對清單執行的查詢。  將會顯示查詢所傳回的記錄計數。 |
| 色彩 | 長條的色彩。 |
| 名稱和值分隔符號 | 您要將 text 屬性剖析成多個值時的單一字元分隔符號。  如果您指定分隔符號，您可以在 [名稱] 方塊中提供每個欄位的名稱 (以相同的分隔符號分隔)。<br><br>例如，考慮名為 location 的屬性，其包含 Redmond-Building 41 和 Bellevue-Building12 等值。  您可以針對 [名稱與值分隔符號] 指定 –，針對 [名稱] 指定 City-Building。  這會將每個值剖析成兩個名為 City 和 Building 的屬性。 |
| 瀏覽查詢 | 當使用者選取清單中項目時所要執行的查詢。  使用 {selected item} 來包含使用者所選項目的語法。<br><br>例如，如果查詢具有名為 Computer 的資料行且瀏覽查詢是 {selected item}，則會在使用者選取電腦時執行 Computer="MyComputer" 之類的查詢。  如果瀏覽查詢是 *Type=Event {selected item}*，則會執行 *Type=Event Computer="MyComputer"*。 |
| **清單** | **> 資料行標題** |
| 名稱 | 顯示於清單第一欄頂端的文字。 |
| 值 | 顯示於清單第二欄頂端的文字。 |
